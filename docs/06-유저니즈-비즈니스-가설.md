## 3. 니즈 → 해결 → 가치 매핑

**"우리 서비스가 어떤 점을 해결하고, 왜 도움이 되는가"** — 3축 기능과 니즈의 대응.

| 니즈(세그먼트) | 우리가 해결하는 것 (3축 매핑) | 왜 도움이 되는가 (가치 논리) | 근거 · 상태 |
|---|---|---|---|
| "통할지 모르겠다" (S1) | **① 진단 리포트**: 일본 문법 진단 점수 — 표현 규제 체크 + 한일 문법 차이(단일제품 소구·정보 밀도·카루셀 구조) + Before/After 재작성 샘플 | "번역이 아닌 재설계"를 **측정 가능한 항목**으로 바꿔줌. 추상 슬로건이 아니라 "12개 항목 중 3개 충족" 같은 점수 → 대행 견적보다 싸고 빠른 사전 판단 | 한일 문법 차이 ⚠️ 리서치 §2 · Before/After가 최강 자산(12/20) 🟡 시뮬2 §2 |
| "위반인지 즉시 알고 싶다" (S4) | **① 진단 + 상시 체커 모듈**: NG/OK 표현쌍 DB 기반 검출 + 대체 표현 제안 (→ [[research/01_MVP스펙_약기법카피검수_KCopyGuard]] 승격) | 위반 책임과 과징금(매출 4.5%)은 **브랜드가 진다** → 보험 가치. 근거가 법 조항이라 **AI 불신 우회**(모델 의견이 아닌 검증 가능한 룰) | 규제 ✅ 리서치 §7·8 · 훅 강도 🟡 시뮬1 §4-8 · AI 거부층 6명 존재 🟡 시뮬2 §4-6 |
| "인스타에서 안 팔린다" (S1·S2) | **① 진단의 KPI 재정의**: 인스타는 전환 채널이 아니라 **인지·저장 → 리뷰 검증 연결** 채널임을 진단에 내장 | 일본 구매 여정(인스타 1착 → @cosme 2착 → 전환은 YouTube·리뷰)은 한국 브랜드 대부분이 모르는 사실 → "저장률·리뷰 연결을 설계하라"는 처방 자체가 차별 가치 | 구매 여정 ✅ 리서치 §10 (3표 검증) |
| "만들 손이 없다" (S1·S3) | **② 마케팅 스튜디오**: 일본 문법 템플릿 내장 생성 — 카루셀 図解 5단 구조, 그리드 반복 패턴·컬러 75/25/5, 置き画 스타일, 퍼스널컬러 축, 발매일·가격 명기형 | 일본 콘텐츠 문법이 **정형화되어 있어 템플릿화 가능** — 에이전시가 사람으로 하던 일 중 규칙 기반 부분만 SW화. 금지 포지션(범용 생성기)과의 경계 = 일본 문법 룰셋 내장 | 문법 정형성 ⚠️ 리서치 §2·3 · 에이전시 90%·SW 부재 ✅ 시장 리서치 · 스튜디오 훅 🟡 시뮬2 (P16~18) |
| "배우지 않고 결과물만" (S3) | **② 스튜디오의 전달 모드**: "직접 쓰는 툴" 외에 "초안이 도착합니다" 배달형 옵션 | 같은 엔진, 전달 방식만 분리 → 지불 여력 있는 위임형 회수. 랜딩에서 "누가 손을 움직이는지" 명시 | 이탈 관찰 🟡 시뮬2 §4-4 · 회수 가능성 ❌ (H4) |
| "리뷰가 안 쌓인다" (S2) | **③ 운영**: 리뷰 메커니즘 산출물 — 기프팅 브리핑 키트(NG 표현 요약 + PR 표기 가이드), UGC 리포스트 운영안, 리뷰 유도 문안, 샘플링→나노 인플루언서 퍼널 설계 | 확산 정형 퍼널(샘플링→나노→UGC→랭킹)이 외부 검증됨 + 스테마 표기 관리는 브랜드 법적 의무라 **가이드 자체가 필수품** | 퍼널 구조 ⚠️/✅ 리서치 §4·8 · 수요 🟡 시뮬2 §4-5 · 상품 형태 ❌ (H5) |
| "언제 뭘 올려야 하나" (S1·S3) | **②·③ 시즌 캘린더 내장 기획**: 메가와리 연 4회(3·6·9·11월) + 코프레(8월 해금~11월 발매)·UV(4~7월)·가을 신색 사이클 기반 콘텐츠 캘린더 | 콘텐츠 수요가 **반복적·예측 가능** → "다음 시즌 대비"가 매달 존재 = 구독 갱신의 자연 근거 | 시즌 사이클 ✅ 리서치 §11 · 구독 전환 ❌ (H3) |

**우리가 해결하지 않는 것 (비타겟 — 금지 포지션과 정합)**
- 물류·통관·인허가 대행 (에이전시·전문사 영역), 광고 집행 대행, 범용 번역, 종합 운영 대행, AI 영상 생성 → [[00-positioning]] 금지 포지션 6종
- S2의 페인 중 "리뷰 대량 확보 보장" — 우리는 구조 설계·산출물까지. 보장형 약속은 스테마 규제 리스크

| **경쟁 우위(해자)** | **축적형 데이터 자산**: NG/OK 표현 DB(법 기반, 변동 적음) + 일본 문법 체크 항목 + 시즌 캘린더 + 벤치마크 계정 DB. 에이전시는 사람에 묶여 못 만들고, 범용 생성툴은 일본 문법 룰셋이 없음 | ❌ 추론 |

**수익 퍼널의 논리 (시뮬레이션이 그려준 행동 패턴)**
1. **Free = 약기법 간이 체커(승격안)** — URL/캡션 입력 → 위반 표현 N건 + 조항 근거. "결과물을 보고 판단하겠다"(17명)에 대한 유일하게 정직한 답이자, 결과 자체가 Report 영업 🟡
2. **Report 30만 = 관문** — 가격 저항 사실상 소멸(17/20 고려). 페르소나 전원이 "Report 보고 구독 판단" 경로를 스스로 그림 🟡
3. **Growth = 시즌 구독** — 장벽은 금액이 아니라 "결과물 확인 전 월정액" 구조 → Report 비용 공제 브릿지 + "다음 메가와리·코프레를 놓치지 않는 기획"으로 갱신 근거 부여 ❌
4. **증거 전략** — 가상 스탯 금지(시뮬2 최대 이탈 트리거, 15명 불신). 대신 ①샘플 리포트 실물 ②벤치마크 분석 공개 ③채점 기준 공개 🟡

## 5. 핵심 가설 백로그 & 검증 계획

| # | 가설 | 현재 증거 | 검증 방법 | 반증 조건(폐기 기준) |
|---|---|---|---|---|
| **H1** | Report 30만이 전환 관문이다 — Free→Report 전환이 퍼널의 병목이자 허브 | 시뮬2 17/20 수용 🟡 | 실고객 인터뷰 5~10명 + 유료 제안 (MVP 실행안 §9) | 인터뷰에서 30만 지불 의사 0~1명 / "샘플 보고도 못 믿겠다"가 지배 |
| **H2** | 약기법 체커가 최강 무료 웻지다 — 즉시 가치 + 신뢰(법 근거) + Report 영업을 동시 해결 | 훅 신호 🟡 시뮬1 · 규제 실재 ✅ | 3차 시뮬(체커 버전 랜딩) → 랜딩 A/B(체크리스트 vs 체커) | 체커 사용→Report 전환이 체크리스트 대비 우위 없음 |
| **H3** | 구독(Growth)의 갱신 근거는 시즌 캘린더다 | 시즌 수요 반복성 ✅ · 전환 ❌ | UT(8/1~3)에서 "다음 시즌 기획" 반응 관찰 → 베타 고객 1~3곳 갱신률 | 베타 고객이 시즌 기획을 안 쓰거나 1회성 소비 후 이탈 |
| **H4** | 위임형(S3)은 "초안이 도착합니다" 전달 모드로 회수 가능하다 | 이탈 관찰만 🟡 | 3차 시뮬에 전달 모드 문구 반영 (P15·16·17 점수 변화) | 문구 수정 후에도 위임형 점수 정체 → 별도 서비스 라인 필요 판단 |
| **H5** | S2(운영 정체)는 리뷰 메커니즘 산출물로 전환 가능하다 | 니즈 ✅ · 해결책 ❌ | 리뷰 메커니즘 1블록(인과 서사+산출물 목록) 추가 후 3차 시뮬 P09·11·13 | 서사 추가에도 "그래서 리뷰가 쌓이냐" 불신 지속 → S2는 2차 타겟으로 강등 |
| **H6** | 진단의 신뢰는 "채점 기준 공개 + 샘플 실물"로 확보된다 (가상 증거 불필요) | 가상 증거 역효과 확인 🟡 시뮬2 §4-1 | P0 반영(스탯 제거+샘플 공개) 후 동일 20명 3차 시뮬 | 3차에서도 평균 2점대 정체 → 신뢰 장벽이 증거가 아닌 다른 곳에 있음 |
| **H7** | 일본 문법 룰셋은 축적형 해자가 된다 | SW 공백 ✅ 시장 리서치 · 추론 ❌ | 개발하며 룰셋 구축 비용 실측 → 경쟁사(Beauty Intelligence 등) 대비 차별 유지 관찰 | 룰셋 구축 비용이 스프린트 예산 초과 / 범용 LLM만으로 동일 품질 재현 가능 |
| **H8** | 인스타 KPI 재정의(저장·리뷰 연결)가 진단의 지적 권위를 만든다 | 여정 구조 ✅ · 고객 반응 ❌ | 인터뷰에서 "인스타는 파는 곳이 아니다" 설명에 대한 반응 관찰 | "이미 알고 있다" 반응 다수 → 차별 포인트에서 제외 |

**검증 수단 일정 매핑**
- **3차 시뮬레이션** (P0 반영 후, ~7/10): H2·H4·H5·H6 — 카피/구조 반응은 시뮬로 선별 가능
- **실고객 인터뷰 5~10명** (병행, MVP 실행안 §9 4주차): H1·H8 — 가격·인식은 합성으로 더 못 파는 영역
- **UT** (8/1~8/3): H3 + 3축 완주율·신뢰 (02-product-spec 검증 기준)
- **베타 운영** (발표 후): H3·H7 장기 확인

## 6. 기획 시사점 → 개발 스펙 연결 (7/11~)

진단 리포트 스펙([[02-product-spec]] TODO)에 주는 입력:

| 스펙 요소 | 이 문서의 근거 |
|---|---|
| 진단 항목 = ①표현 규제(NG/OK 검출) ②일본 문법 체크(단일제품 소구·정보 밀도·카루셀 구조·그리드) ③KPI 재정의(저장·리뷰 연결 설계) | §3 매핑 · 리서치 §2·7·10 |
| 산출물 = 문법 점수 + NG/OK 재작성 샘플(Before/After 형식 — 최강 자산) + 시즌 캘린더 발췌 | 시뮬2 §2 (12/20) · H6 |
| 입력 = URL·인스타 계정·제품 분류(화장품/의약외품 — 허용 표현이 다름) | 리서치 §7 (카피 이원화) |
| 신뢰 장치 = 채점 기준 공개 + 조항 인용. 가상 수치 금지 | 시뮬2 §4-1 · H6 |
| 무료/유료 경계 = 체커(위반 검출)는 Free, 재작성·페르소나·USP는 Report | §4 퍼널 · H2 |

우선 구축할 데이터 자산 순서: **약기법 NG/OK 표현쌍 DB(소스 명확·변동 적음) → 시즌 캘린더(리서치 완료) → 문법 체크 항목(리서치 기반 초안) → 유행어·감성어 크롤링(가장 무거움, 후순위)**.
