# 04 · 로드맵

> 관련: [[02-product-spec]] · [[05-open-questions]]

## Phase 1 — 랜딩페이지 (현재)

**목표:** 광고 → 랜딩 → 무료 "5분 진단" 신청으로 **수요·메시지 검증**.

- [ ] 랜딩 1페이지: 문제 자극형 히어로 + 제안 설명 + 신청 CTA
- [ ] "5분 진단" 신청 폼 (브랜드/URL/인스타/목표/고민)
- [ ] 신청 완료(thanks) 페이지
- [ ] 리드 저장 방식 결정 → 외부 폼서비스 vs 자체 API (열린 결정)
- [ ] 광고 문구 A/B/C/D 중 초기 테스트 세트 선정 (→ jp-localizer/pm-planner)
- [ ] 배포 (Vercel 등)

**완료 기준:** 광고를 붙였을 때 클릭률·신청률·무료 진단 신청 수를 측정할 수 있는 상태.

## Phase 2 — 한 사이클을 경험하는 작동 서비스 (범위 미정)

**목표:** 신청자가 "무료 5분 진단"의 한 사이클(입력 → 결과 리포트 수령)을 **실제로 경험**하는 최소 작동 서비스.

- [ ] 한 사이클의 범위 정의 (pm-planner 주도, `/spec`)
  - 후보: 입력 폼 → (AI 보조) 미니 리포트 자동 생성 → 결과 페이지/이메일 전달
- [ ] 진단 로직/프롬프트 설계 (jp-localizer + backend-dev)
- [ ] 결과 리포트 UI (designer + frontend-dev)
- [ ] 데이터 저장·관리 필요 여부 결정 (backend-dev)

> Phase 2는 별도 기획 세션에서 범위를 확정한 뒤 착수한다. 지금은 구조만 확장 가능하게 열어둔다.

## 원칙
- 각 Phase는 **검증 가능한 완료 기준**을 갖는다.
- 툴을 직접 만들지 않는다 — 기존 AI 툴을 조합하는 컨시어지 방식 유지 ([[00-positioning]]).
