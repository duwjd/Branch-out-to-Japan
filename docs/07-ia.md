## 1. 문서 목적 · 범위 · 전제

**목적** — 흩어진 축별 스펙·랜딩 시안·Figma 컴포넌트를 하나의 정보구조로 통합해, 이후 화면 설계(`/design-page`)·라우팅·개발의 상위 기준을 확정한다.

**범위** — 전체 서비스 IA: **공개 퍼널**(랜딩·무료 약기법 체커·샘플 리포트·요금) + **앱 영역**(진단 리포트·마케팅 스튜디오·운영·마이페이지·브랜드 자산).

**전제 (2026-07-09 기준)**
- 앱 코드는 **아직 없음** — 문서 + 정적 HTML 시안(`persona-simulation/landing/`) + Figma(`design/DESIGN.md`) 단계. 본 문서는 **설계 지도**이지 구현 명세가 아니다.
- 성숙도 차이: **① 진단 리포트 = 개발 착수용 스펙 확정 v2**([[specs/01-report-spec]]) · **② 스튜디오 = 썸네일 변환기만 파이프라인 확정**([[specs/02-thumbnail-converter-spec]], API 실검증 전) · **③ 운영 = 방향만 정의, 스펙 미착수**.
- 페르소나 근거는 **합성 페르소나 0차 검증**(실고객 인터뷰 전까지 가설, [[research/simulations/리포트샘플-페르소나-검증]]).
- 커머셜(결제·구독) 서비스 반영 방식은 **개발·UT(8월) 이후 확정** → IA에는 자리만 두고 "미확정" 표기.

## 2. IA 설계 원칙 (근거에서 도출)

1. **① 진단 리포트 = 전환 허브 (중심 노드).**
   무료 약기법 체커가 **in-link**(→ 리포트 업셀), 리포트 블록9·블록0의 고정가 퍼널표가 **out-link**(→ ②·③). 시뮬 전원이 "Report 먼저 → 결과 보고 → 구독 판단" 경로를 스스로 그렸다([[02-product-spec]] 퍼널). IA의 무게중심을 리포트에 둔다.

2. **공통 엔티티 "브랜드 프로필" 위에 3축을 모듈로 배치.**
   3축은 독립 제품이 아니라 **하나의 브랜드 자산을 공유**한다. 특히 **약기법 판정 엔진은 ①리포트와 ②썸네일 검수 게이트가 재사용**([[specs/02-thumbnail-converter-spec]] "리포트 ① 파이프라인의 판정 로직 재사용"). 데이터·자산이 브랜드 단위로 순환하도록 설계한다.

3. **신뢰 IA 3축: 고정가 · ~~사람검수 우선~~ · 실물 샘플.**
   - **고정가** — 견적 왕복 거부가 강함. 다음 단계 가격을 표로 노출(진단 30만 → 스튜디오 월 20만·첫 달 진단비 공제).
   - ~~**사람검수 우선** — 'AI 자동' 전면 배치는 AI 경계층 6인 첫인상 감점. "일본향 전문가가 확정, AI가 속도를 낸다"로 **사람 먼저** 프레이밍.~~
     > 🔴 **2026-07-16 붕괴.** 검수 단계가 제품에서 제거되어 **이 축을 뒷받침할 사람이 없다**. "전문가가 확정"은 이제 사실이 아니므로 카피로 쓸 수 없다. 남은 것은 **근거 공개**(채점 기준·내 문장·코퍼스 근거·조항 각주)뿐이며, 이것이 AI 경계층 6인에게 사람 서명만큼 작동하는지는 **검증되지 않았다**. 3축 중 1축이 비어 있는 상태 → [[decisions/DECISIONS]] 🔴 · [[specs/01-report-spec]] v3.
   - **실물 샘플** — 가상 통계·익명 각주는 **최대 이탈 트리거**([[02-product-spec]] 증거 원칙). 샘플 리포트 실물·채점 기준 공개·벤치마크 분석으로 대체.

4. **사용자 모드 분기: 직접 쓰는 팀(툴형) vs 맡기는 팀(배달형).**
   S3(위임형)는 "생성기"·"직접 쓰는 스튜디오" 표현에서 "결국 내가 또 배워야 하네"로 이탈. IA/카피는 "**자산 연결만 하면 초안→검수→완성본이 도착**, 배울 것 없음"을 명시해 배달형 경로를 함께 제공한다.

5. **금지 포지션(6종) 경계 유지.**
   AI 광고영상 생성 / 쇼츠 자동변환 / 채널 분석기 / 예약 발행툴 / 단순 번역 / 종합 대행으로 흐르는 화면·카피는 교정([[00-positioning]]). 모든 생성 화면은 "메시지 재설계"가 앞선다.

## 3. 사용자 & 진입 세그먼트

핵심 타깃: **3~15인 규모 인디 코스메틱 브랜드의 마케터 겸 대표**(겸직 소수). "일본어 번역은 되지만 일본 고객에게 먹히는 표현은 모름"([[research/archive/페르소나-검증형-MVP-실행안]]).

| 세그먼트 | 정의 | 핵심 페인 | 우선순위 | 주 진입점 |
|---|---|---|---|---|
| **S1 입점 준비/직전** | 진출 준비 중, 미입점 | "번역만으로 되는지 확신 없음 · 어디부터 손댈지 막막" | **1차 타깃** | 랜딩 → 무료 체커 → **리포트** |
| **S2 입점 후·매출저조** ★승격 | 입점했으나 매출·리뷰 정체 | "리뷰(口コミ)가 안 붙어 랭킹이 안 오름" | **검증된 유료 전환** | 랜딩(운영) → **S2 재진단 리포트**([[specs/01-report-spec-postentry]]) → Growth(리뷰 운영) |
| ┗ S2-a / S2-b | 리뷰 있음 / 리뷰 0·희박 | 자기 리뷰 실측 / 경쟁사 리뷰 마이닝 | — | 재진단 분기 키(reviewUrl) |
| **S3 리소스 부재·위임형** | 만들 손·감 없음 | "직접 쓰는 툴이면 또 배워야 함 → 이탈" | 툴 포지션과 충돌 | 랜딩(배달형 카피) → 리포트 → 스튜디오 |
| **S4 규제 민감** | 의약외품·기능성 | "薬機法 경계 모름 · 상시 체크 니즈" | 약기법 훅 최강 | **무료 약기법 체커**(상시) → 리포트 |
| **S0 미진출 조사** | 대조군 | — | 타깃 아님 | 무료 체커(리드만) |

**역할축 — 품의형(결재 필요).** 자기가 결정권자가 아니라 상사에게 결재를 올리는 담당자(예: 해외사업 담당). "상사에 그대로 들이밀 수 있다"는 **품의용 표지(리포트 블록0)** 가 전환 장치. IA는 결재용 산출물(표지·PDF) 동선을 별도로 보장한다.

## 5. 글로벌 내비게이션 모델

- **공개 영역** — 랜딩 상단 앵커 내비(서비스 / Before·After / 요금 / FAQ) + 상시 CTA `무료 진단 시작`. 전환 목표 단일화: **"무료 진단 신청"**.
  - ⚠ **재정리 필요(2026-07-16 v4):** 두 진단 모드 모두 **유료**이고 무료 = **샘플**(표지 전체 + 요약 일부(종합점수 가림) + 페르소나 맛보기 — 두 모드 공통)로 재정의됐다([[specs/01-report-spec]] §2 v4). 무료 진입점이 둘(체커·샘플)이 된 랜딩 동선·"무료 진단" CTA 문안 정리는 **미결** → [[decisions/DECISIONS]]. 이 항목의 CTA 서술은 그 재정리 대상이라 v4에 맞춰 고치지 않고 보존한다.
- **앱 영역** — 상단 바에 **브랜드 프로필 스위처**(작업 대상 브랜드 컨텍스트 고정) + 사이드/탭에 3축(① 리포트 · ② 스튜디오 · ③ 운영) + 마이페이지. 어느 축에서 작업하든 "지금 어느 브랜드"가 유지된다.
- **축 간 전환 동선** — 리포트 결과 화면의 블록9 "다음 단계" 카드가 **② 스튜디오로 직접 링크**(고정가 퍼널). 스튜디오 생성 자산은 **③ 운영의 자산 라이브러리에 자동 적재**. 즉 3축은 브랜드 프로필을 축으로 순환.
- **무료→로그인 경계** — 무료 약기법 체커는 비로그인 사용 가능(즉시 가치), 리포트 발행 시점에 회원가입·브랜드 프로필 유도.

## 6. 화면 인벤토리 (축별 상세)

각 화면: **목적 / 핵심 컴포넌트 / 입력·출력 / 인접 링크 / 성숙도**.

### 무료 약기법 체커 (공개 · in-link)
- 목적: 즉시 가치 증명 + 리포트 영업 훅. 근거가 "법"이라 AI 불신을 우회.
- 입력: URL 또는 캡션 텍스트 → 출력: 위반 표현 후보 + 조항 근거 + [본 리포트 업셀 지점].
- 인접: → ① 리포트. 성숙도: 승격안(별도 모듈, [[specs/01-report-spec]] §7 Out).

### ① 진단 리포트  (성숙도: 확정 v4 — 입력 브랜드 우선·두 진단 모드)
| 화면 | 핵심 컴포넌트 | 입력 → 출력 |
|---|---|---|
| **진단 입력폼** | 2단 섹션 — **브랜드 필수**(브랜드명 · 포지셔닝: 택소노미 태그 1~5(`lib/engine/rules/positioning.ts` 16종)+자유 서술 · 카테고리) → **제품 전부 선택**(분류·제품명·성분·가격·상세 콘텐츠 URL/텍스트 단일 필드 = **정밀도 게이지**), 50자 하드게이트(버튼 잠금)·200자 소프트선 — 게이트 단일 정의 `lib/engine/rules/gates.ts` | 브랜드 필수 3종 [+제품 선택 필드] → 제출 경계에서 진단 모드 확정: `brand`(콘텐츠 없음) / `brandProduct`(콘텐츠 있음) |
| **처리 로딩** | 진행 표시 (LLM 4콜 파이프라인) | — |

## 7. 공통 엔티티 · 데이터 모델 (개념 수준)

```
브랜드 프로필 (공통 엔티티)
 ├── 상품
 │     └── 채널 상정 (Qoo10 / 라쿠텐 / 아마존JP)
 │           └── 상세페이지 카피 (문장 단위 K1 … Kn)
 ├── 생성 자산
 │     ├── 진단 리포트 (블록0~8 · PDF)
 │     └── 썸네일 (스타일 A~H · 플랫폼별)
 └── 시즌 캘린더 (메가와리 3·6·9·11월 · 코프레 · UV 4~7월)
```

**작업 단위 3층** — IA는 세 층의 반복 리듬을 모두 담는다:
1. **상품 · 문장 단위** (진단) — 상세 카피를 문장(K1~Kn)까지 감사·재작성.
2. **채널 × 스타일 단위** (제작) — 플랫폼별·스타일별 반복 생성.
3. **시즌 단위** (운영) — 시즌 사이클이 구독 갱신의 반복 리듬.

**데이터 자산 연결** — `data/processed/`: `product-catalog.jsonl`(제품 카탈로그) · `detail-ocr.jsonl`(상세 OCR·벤치마크 근거) · `sns-lexicon.csv`(유행어 사전) · `thumbnail-style-labels.jsonl`·`thumbnail-style-prompts.json`(썸네일 스타일·프롬프트 팩).

> **상세 데이터 플로우(개발용)** — 이 개념 모델을 "입력 → 가공(LLM 콜 계약) → 출력 → 저장 엔티티"까지 구체화한 문서는 [[08-data-flow]] (엔티티 필드·상태 머신·LLM 요청/응답 스키마·화면↔데이터 매핑).

## 8. 핵심 유저 플로우

**(a) Free → Report 전환**
무료 약기법 체커(URL/캡션) → 위반 검출 → "왜·어떻게 고칠지 + 전체 문법 진단 + 재작성 = Report" 업셀 → 회원가입·브랜드 프로필 → 리포트 발행.

**(b) Report → Studio → 운영 (고정가 퍼널)**
리포트 블록9 "다음 단계" 카드(① 브랜드 진단 (가격 미정) → ② 브랜드+제품 진단 30만 → ③ 스튜디오 월 20만·첫 달 진단비 공제 — 운영은 스튜디오 구독 내 확장) → 스튜디오 진입 → 생성 자산이 운영 라이브러리에 적재.

**(c) 3축 한 사이클 완주 (UT 검증 단위)**
입력 → 리포트 → 콘텐츠 생성. [[02-product-spec]]의 UT 성공 기준과 동일 — IA는 이 단일 여정을 끊김 없이 관통해야 한다.

**(d) 품의형: 결재용 산출 동선**
리포트 → 블록0 품의 표지 + PDF 내보내기 → 상사 결재 → 재방문·구독. "견적 불투명 이탈"을 해소하는 전용 경로.
