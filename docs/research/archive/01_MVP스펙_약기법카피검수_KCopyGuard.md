## 1. 스코프 (MVP = 약기법만, 그 외 전부 나중)

### ✅ MVP 포함 (v1)
1. 일본어 카피 붙여넣기 → 문장/구절 단위 위반 하이라이트
2. 위반 유형·근거(어떤 규제/례) 표시
3. 리스크 등급: 🔴위반 / 🟡주의 / 🟢안전
4. 위반 구절별 **수정 대안 카피 3안** 자동 제시
5. 제품 카테고리 선택(스킨케어/색조/헤어/의약외품 후보) → 카테고리별 룰 적용
6. 검사 결과 요약 리포트(복사/PDF)
7. 회원가입·로그인·사용량 카운트·구독 결제

### ❌ MVP 제외 (v2+로 미룸)
- 이미지/동영상 OCR 검사, @cosme/Qoo10 자동 크롤링 연동, 팀 협업/워크플로우, 성분 규제 판별(2위 아이디어), 다국가(EU/미국) 확장, 브라우저 확장/피그마 플러그인.

## 2. 핵심 로직 — "룰 + LLM 하이브리드"

정확도와 비용을 잡는 2단 파이프라인. **LLM 단독 금지**(환각·근거 부재 위험).

```
입력 카피(JP)
   ↓
   ├─ 1단계: 룰 엔진 (결정적, 무료, 빠름)
   │    · NG워드/금지표현 사전 매칭 (정규식 + 형태소)
   │    · 예: 「シミが消える」「アンチエイジング」「美白」(범위 이과정)「治す」「若返り」
   │    · 결과: 후보 위반 + 위치(offset)
   │
   ├─ 2단계: LLM 판정 (문맥화, 룰이 못 잡는 뉘앙스)
   │    · claude-haiku-4-5 (저비용) 로 1차, 경계 케이스만 claude-sonnet-5 승급
   │    · 입력: 카피 + 카테고리 + 적용 규제 발췌(RAG) + 룰 결과
   │    · 출력(JSON): [{span, severity, rule_id, reason, alternatives[3]}]
   │    · 근거 규정을 반드시 인용하도록 강제(citation) → 환각 최소
   │
   └─ 병합·중복제거 → 하이라이트 렌더링 + 리포트
```

**규제 DB(핵심 자산)**: 화장품 표시·광고 관련 약기법 조문, 후생노동성 「화장품등의 적정광고 가이드라인」, 화장품공정거래협의회 자율규약, 대표 NG/OK 표현 대응표를 구조화(rule_id, 카테고리, 패턴, 근거, 안전 대안 예시). → 초기 200~400 룰로 시작, 사용 로그로 지속 확장.

> ⚠️ **면책 필수**: "본 도구는 참고용이며 최종 법적 판단·인허가를 대체하지 않습니다." 모든 리포트·화면에 명시.

## 4. 기술 스택 (1인 개발 최적)

| 레이어 | 선택 | 이유 |
|--------|------|------|
| 프론트 | **Next.js(App Router) + TypeScript + Tailwind + shadcn/ui** | 표준 단일 레포, 배포 간단 |
| 하이라이트 에디터 | textarea 오버레이 or Tiptap(경량) | 인라인 마킹 |
| 백엔드 | Next.js Route Handlers (서버리스) | 별도 서버 불필요 |
| DB | **Supabase(Postgres)** | 인증·DB·RLS·스토리지 통합 |
| 규제 DB | Postgres 테이블 + `pgvector`(RAG 근거 검색) | 룰+임베딩 |
| LLM | **Anthropic claude-haiku-4-5**(기본) / **claude-sonnet-5**(경계 케이스 승급) | 비용/정확도 균형 |
| 결제 | **Stripe** (또는 국내 대상이면 Paddle/토스페이먼츠) | 구독 |
| 배포 | **Vercel** | Next.js 네이티브 |
| 분석 | PostHog | 퍼널·채택률 |

**LLM 호출 표준(참고)**: 모델 ID `claude-haiku-4-5-20251001` / `claude-sonnet-5`. 출력은 tool use(structured output)로 JSON 스키마 강제, 규제 인용 필드 필수. 프롬프트 캐싱으로 규제 컨텍스트 재사용 → 비용 절감.

### 데이터 모델(요지)
```
users(id, email, plan, usage_count, period_start)
rules(id, category, pattern, type, severity, legal_basis, safe_examples[])
checks(id, user_id, category, input_text, result_json, risk_level, created_at)
subscriptions(user_id, stripe_id, plan, status, current_period_end)
```

### 비용 감각 (개략)
- 검사 1건 → 입력 수백 토큰 + Haiku 출력 → 건당 **약 $0.001~0.005**.
- Pro($79/월·500건) 원가 ~$2.5 → **마진 95%+**. LLM 비용이 위험 아님.

### 검증 게이트 (만들기 전에 확인)
- W0(사전): 운영대행사 3~5곳 인터뷰 → "카피 감수에 얼마·얼마나 자주 쓰는지" 확인. **지불의향 없으면 2위(인허가 체크리스트)로 피벗.**
- W8 게이트: 실측 정밀도 <80%면 출시 보류(오탐이 신뢰 파괴).
