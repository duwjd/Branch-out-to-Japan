# 호스팅·배포 결정 — Vercel Hobby + Supabase Free

> 2026-07-24 · 상태: **확정** · 배경: UT(8/1~3) 전 배포 필요(08 §8-D1), 호스팅 플랫폼은 문서상 미결이었음
> 관련: [[../11-deploy-spec]] · [[../08-data-flow]] §6.2 · [[../10-implementation-status]] §5 · [[DECISIONS]]

## 결정

1. **앱 호스팅 = Vercel Hobby(무료)** — GitHub `duwjd/Branch-out-to-Japan`(public) 연동, main push → 자동 프로덕션 배포.
2. **저장 = Supabase Free** — Postgres(스토어 11테이블) + Storage(private 버킷 `files`). 08 §6.2 기본안·기 구현 코드(`lib/db/supabaseStore.ts`)를 그대로 실전환.
3. **실행 모델 = `after()` + `maxDuration 300` 유지, 큐 미도입** — Vercel Fluid Compute(신규 기본 on)로 Hobby도 함수 300초 실행 가능. 파이프라인 실측 예상 2~3분이 예산 안이라 08 §6.2 큐 fallback은 발동 조건 미충족. 10 §5 #7("서버리스 배포 시 큐 필요")은 이 결정으로 해소.
4. **인증 메일 = `AUTH_MAIL_MODE=devlink` 임시 모드** — 실메일 미구현 상태에서 운영 가입이 전원 차단되는 문제(mailer가 production에서 링크 미반환)를, 폐쇄 UT 한정으로 인증 링크 화면 노출로 우회. 실메일(Resend)은 P1.

## 배경 / 대안 비교

목표: **간편 + 무료**. Next.js 16 + `after()` 백그라운드 잡 + Supabase 조합 기준.

| 후보 | 판정 | 이유 |
|---|---|---|
| **Vercel Hobby** | **채택** | Next 16·`after()` 네이티브, Fluid 300초로 LLM 파이프라인 수용, GitHub 연동만으로 CI/CD 완성(설정 제로) |
| Cloudflare Workers | 탈락 | OpenNext 어댑터 경유 — 설정 복잡 + `after()` 동작 검증 부담. "간편" 위배 |
| Netlify Free | 탈락 | 무료 함수 실행시간 짧고 background functions 유료 |
| Render / Railway | 탈락 | Render 무료는 15분 슬립(첫 접속 30초+ — UT 첫인상 최악), Railway는 크레딧 소진형 |

## 감수하는 리스크 (명시)

- **Hobby 비상업 한정** — 무수익 폐쇄 UT는 가능하나, **유료 고객을 받는 시점 = Pro($20/월) 전환 트리거**. 과금 전 팀 결정 필요.
- **리포 public 전제** — private org 리포는 Hobby 연동 불가. private 전환 시 Pro 또는 GitHub Actions 우회(11 §7).
- **Supabase 7일 무활동 pause** — UT 직전 접속 확인으로 예방(11 §6).
- `devlink` 모드는 인증 링크가 화면에 노출되는 임시 상태 — 폐쇄 UT 한정 수용, 실메일 도입 시 제거.
