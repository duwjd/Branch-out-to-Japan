# specs/ — 개발 착수용 스펙 (3축 넘버 규칙)

넘버는 **작성 순서가 아니라 3축(제품 라인) 기준**이다.

| 넘버 | 축/라인 | 파일 | 상태 |
|---|---|---|---|
| **00** | ⓪ 진입 셸 · 메인(앱 홈) | `00-main/` (UI 기획서·프로토타입) | 초안 (2026-07-20) |
| **01** | ① 진단 리포트 · S1(입점 전) | `01-report-spec.md` | 개발 착수용 확정 v2 |
| **01** | ① 진단 리포트 · S2(입점 후) | `01-report-spec-postentry.md` | 초안(01의 델타 · 신규블록 A·C) |
| **01** | ① 진단 리포트 · 깊이 샘플 | `01-report-sample-cica-ampoule.md` | 30만 가치 증명 기준선 |
| **02** | ② 마케팅 스튜디오 | `02-thumbnail-converter-spec.md` | 파이프라인 확정(API 실검증 전) |
| **03** | 커머셜(축 횡단) | `03-구독플랜-재구성-제안.md` | 확정(폴리시 수치 조장 대기) |
| **03** | 커머셜 · 계정(로그인·마이페이지) | `03-account/` (UI 기획서·프로토타입) | 초안 (2026-07-20) |
| **04** | ③ 운영 (자산 라이브러리·기업 매칭) | `04-operations/` (UI 기획서·프로토타입) | 초안 (2026-07-20) |

> ① 리포트 라인은 `01-` 접두사로 묶는다(S1·S2·샘플). ② 스튜디오=`02-`, 커머셜=`03-`, ③ 운영=`04-`(자산 라이브러리·기업 매칭 UI 기획 착수, 리뷰 축적 운영 키트 등 구독 실체는 `03-구독플랜-재구성-제안.md` 정본 유지). 제품 축에 속하지 않는 진입 셸(앱 홈)은 축 앞의 `00-`. 계정(로그인·마이페이지)은 구독 플랜 가입 정보를 다루므로 커머셜과 같은 `03-`으로 묶는다.
