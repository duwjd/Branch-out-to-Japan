# 0. 개요

로그인은 소셜·이메일 인증으로 앱에 진입하는 화면이고, 마이페이지는 구독 플랜 가입 정보·계정 정보·결제·브랜드 프로필을 한곳에서 확인하는 화면이다. 두 화면으로 ⓪ 메인 기획서가 "자리만" 잡아 두었던 계정 메뉴(MAIN-01e)와 플로우 맵의 "로그인 · 온보딩(범위 외)" 노드를 실제 화면으로 채운다. 더해, 비회원이 앱을 둘러보다 생성·등록을 시도할 때 뜨는 **로그인 유도(게이트) 모달**을 이 문서 §3에서 한 번 정의하고 각 축 기획서가 이를 참조한다.

인증 수단은 소셜 로그인 3종(구글·카카오·네이버)과 이메일/비밀번호를 병행한다(2026-07-23 개정 — 소셜 3종 단독 확정(2026-07-20)을 대체, `../decisions/DECISIONS.md`). 이메일/비밀번호 가입·로그인은 폐기가 아니라 되살린 축으로, 폼 원안은 `design/wireframes/public-onboarding-spec.md` §5를 계승한다. 구현은 Supabase Auth의 OAuth provider(소셜)와 이메일/비밀번호 인증을 전제한다(`../09-dev-spec.md` §1). ※ 실 인증(OAuth·메일 발송)은 미구현 — 목 세션으로 시연한다(`../10-implementation-status.md`).

구독 플랜의 이름·가격·구성 정본은 `03-구독플랜-재구성-제안.md`(FREE / Report 30만·1회 / Growth 월 20만 / Scale 월 80만)다. 단, 결제 연동(PG)·구독 엔티티·결제 게이트 집행은 미구현이므로(`../10-implementation-status.md`), 마이페이지의 결제·구독 영역은 표시 계약만 정의하고 실데이터 연동 방식은 (미정)으로 남긴다. 화면 속 결제 수단·결제 내역은 전부 목업이다.

PC 기준으로 설계하며 모바일은 반응형 대응만 한다. 로그인은 셸 없는 단독 화면, 마이페이지는 ⓪ 사이드바 셸(00-main 기획서 MAIN-01)을 재사용한다. 온보딩(브랜드 프로필 등록), 이용약관·개인정보처리방침 문서, 결제 연동 실구현, 탈퇴 백엔드 정책(데이터 보존·삭제)은 이번 범위에서 제외한다.

# 3. 로그인 유도 모달 (게이트)

비회원이 앱을 둘러보다 **회원 정보가 필요한 액션**을 시도하는 순간 뜨는 공용 모달. 이 문서에서 한 번 정의하고 ⓪ 메인·온보딩·① 리포트·② 스튜디오 기획서가 이를 참조한다(중복 정의 X). 진입 게이트(로그인 후 열람)에서 **실행 게이트(열람은 자유, 실행 직전에만 유도)**로의 전환이 이 모달의 존재 이유다(2026-07-23, `../decisions/DECISIONS.md`).
