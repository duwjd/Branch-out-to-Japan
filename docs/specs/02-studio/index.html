일본향 썸네일 스튜디오 — 화면 흐름

한국 썸네일 1장 업로드 → 템플릿 8종 중 1개 선택 → 생성. 홈이 곧 생성 퍼널입니다. 제출하면 결과 화면으로 바로 이동하고, 그 화면이 생성중 상태(5단계 진행)로 시작해 완료되면 결과 뷰로 바뀝니다. 홈에서 시작해 전체 플로우를 클릭으로 체험하세요.

이미지 출처 고지. 화면 속 이미지는 성격이 둘로 나뉩니다. 템플릿 프리뷰는 라쿠텐·아마존JP·Qoo10에서 수집한 실측 참고 컷(타사 저작물 · 분석 목적 인용 · 비배포 한정)이고, 화면에 "실측 참고" 라벨로 구분해 둡니다. 업로드 예시·최근 생성 스트립·결과 이미지는 docs/presentation/샘플.png에서 파생한 자체 생성 샘플입니다. 제품컷 자체는 실물 예시이고, 데모 브랜드 HARUON은 오버레이 표기에만 씁니다. 출처·사용 조건은 assets/README.md. 카피·해설 예시는 정본 스펙(../02-thumbnail-converter-spec.md §3 골든 픽스처)에서 치환했고, gpt-image-2.0 실 API 검증은 미실시(스튜디오 주간 7/18~24 예정)입니다.

전체 플로우 — 홈이 곧 생성이다

이전 개편에서는 홈(갤러리)이 허브였지만, 지금은 ①리포트처럼 "입력→처리→결과"의 직렬 구조입니다. 업로드·템플릿 선택은 홈 화면 하나에서 끝나고, 제출은 결과 화면으로 직행합니다. 보관·재열람은 이 플로우가 아니라 별도 노드(③ 운영)가 맡습니다.

1 · 홈(생성)

스튜디오 홈

업로드 1장 + 템플릿 8종 + 플랫폼 추천 + 프루프 게이트. 하단에 최근 생성 4건.

 → 2 · 결과(생성중)

생성중 → 완료

제출 즉시 이 화면으로 직행. 5단계 진행 후 결과 뷰로 crossfade.

 → 2 · 결과

생성 결과 상세

큰 이미지 + 재설계 해설 3단 + PNG 다운로드(실동작).

생성중에도 다른 화면으로 이동할 수 있습니다(처리는 계속됨 — 홈으로 돌아가면 최근 생성 스트립에 생성중 타일이 보조로 뜹니다). 각 화면의 검토용 상태 스위처로 빈/기본/생성중(홈) · 결과/생성중/실패(결과)를 바로 볼 수 있습니다.

보관 · 재열람 ③ 운영 자산 라이브러리 생성한 썸네일 전체를 모아보고 필터링하는 정본 화면입니다. 이 프로토타입 범위 밖입니다.

템플릿 8종 ↔ 플랫폼 매핑

라쿠텐·아마존JP·Qoo10 실측 948장 중 층화 표본 120장에서 역설계한 분류(jp-thumbnail-style-taxonomy §3). 내부 ID는 이 문서 한정 병기이며 화면 라벨은 전부 평문입니다(①리포트 라벨 정책 계승).

결과 화면 상태 3종 — 2-result.html?state=

현 데이터 계약(08-data-flow §6)의 GeneratedAsset에는 status가 없습니다. 생성중 상태가 성립하려면 status 필드(또는 잡 조회) 신설이 필요합니다(기획서 #0 계약 델타).

?state=generating

생성중

제출 직후 착지 상태. 블러+shimmer 이미지 + 5단계 진행 리스트 + 우측 스켈레톤. 완료 시 결과 뷰로 crossfade.

(쿼리 없음)

결과

큰 이미지 + 검수 게이트 + 다운로드 + 재설계 해설 3단. 검수 게이트 통과분만 자산이 됨.

?state=failed

실패

게이트 불통과 사유 + 다시 시도(홈으로 프리필 재진입). 자산으로 저장하지 않음.
