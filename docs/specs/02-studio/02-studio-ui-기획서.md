# 0. 개요

썸네일 스튜디오는 한국 썸네일/제품컷 1장을 입력받아, 일본 고객이 익숙한 썸네일 문법 8종 중 1개로 재설계된 일본향 썸네일을 생성하는 기능이다. 범용 이미지 생성툴이 아니다 — 일본어 카피는 jp-localizer 재설계 산출(약기법 검수 통과분)만 이미지에 들어가고, 실적 배지는 근거 없이는 생성되지 않는다(`../02-thumbnail-converter-spec.md` §0 핵심 원칙).

**입력한 사실만 렌더한다**가 이 화면 전체를 관통하는 문법이다. 실적 배지(HOME-05)·모델컷(HOME-02b)·가격(HOME-05b) 셋 다 같은 규칙을 따른다 — 근거를 입력하지 않으면 그 요소는 **생성되지 않는 것이 기본값**이고, 입력했다면 **입력한 문자열·이미지 그대로만** 들어간다. 지어내기(임의 인물 합성·근거 없는 랭킹·없는 가격)는 프롬프트 진입 단계에서 코드가 차단한다.

화면은 **스튜디오 홈(생성)**과 **생성 결과 상세** 2개로 구성한다. 스튜디오 홈이 곧 생성 퍼널이다 — 원본 업로드·플랫폼·템플릿 선택·제출을 한 화면에서 처리한다. 생성 로딩은 별도 화면이 아니라 **생성 결과 화면의 시작 상태**다 — 제출하면 결과 화면으로 이동해 생성중 상태로 시작하고, 파이프라인 단계를 거쳐 완료 시 그 자리가 결과로 crossfade 전환된다. 생성 중 다른 화면으로 이동해도 처리는 계속된다. 완료를 확인하는 **보조 표면**은 ⓪ 메인 진행 중 작업 패널과 ③ 운영 자산 라이브러리 그리드다 — **홈에는 자산을 되비추는 표면을 두지 않는다**(2026-07-22 최근 생성 스트립 제거). 썸네일 보관·재열람의 정본 위치는 **③ 운영 자산 라이브러리**다(`../04-operations/04-operations-ui-기획서.md` # 1 자산 라이브러리 · # 2 자산 상세) — 스튜디오 홈은 생성만 담당하고 갤러리·필터·재열람 기능을 두지 X.

화면 라벨은 전부 평문이다. 템플릿은 "클린 스튜디오 단독컷" 같은 이름으로만 표기하고 내부 스타일 ID(A~H)는 화면에 노출하지 X — ① 리포트의 라벨 정책(REPORT-14)을 계승한다.

이 화면이 성립하려면 데이터 계약 델타가 필요하다(`../../08-data-flow.md` §6). (1) `GeneratedAsset.status`(generating/done/failed) — 결과 화면 생성중 시작 상태가 생성 시작 시점의 레코드를 전제한다. (2) `explanationJson`(styleReason·copySlots·krElementMap) — 결과 상세의 재설계 해설은 파이프라인 ①~④단계 산출인데 구 계약은 `promptUsed`·`gateResult`만 저장했다(RESULT-02·03). **(1)·(2)는 1차 개발에서 반영 완료.**

**2026-07-22 추가 델타 3건** — 이번 개정의 전제다. (3) `modelImagePath`(모델컷 fileId) · (4) `modelConsent`(사용 권한 동의 여부 — 자산 레코드에 함께 보존) · (5) `promoInput`(세트명·판매가·통상가·통상가 실적 확인 플래그·할인율·GIFT·한정 칩·각주). 생성 호출은 모델+카피형에서 **[제품컷, 모델컷] 2장 배열**로 나간다(`../02-thumbnail-converter-spec.md` §2-⑥).

**2026-07-22 제품컷 소스 델타** — (6) `productImageSource`(upload | brandAsset) · (7) `productId`(brandAsset일 때 참조한 제품 — ③ `Product` 엔티티 신설 제안 전제, `../04-operations/04-operations-ui-기획서.md` BRAND-09). 제품컷을 매번 올리는 대신 ③ 브랜드 관리에 등록한 제품컷을 골라 쓴다(HOME-02). 파이프라인에 넘어가는 것은 어느 경로든 **이미지 1장**이며, 자산에는 제출 시점 이미지가 스냅샷으로 남으므로 이후 제품 삭제·교체가 소급되지 X. 이 소스 선택은 **생성 입력**이지 자산 재열람 표면이 아니다 — "홈에 자산을 되비추는 표면을 두지 않는다"(HOME-07)는 완료 자산 갤러리를 금지한 규칙이고, 브랜드가 등록해 둔 입력 재료를 고르는 것은 그 규칙의 대상이 아니다.

앱 셸(사이드바 전역 내비·브랜드 스위처·KPI 위젯)은 ⓪ 메인 기획서 MAIN-01을 정본으로 따른다(2026-07-20 정합). PC 기준으로 설계하며 모바일은 반응형 대응만 한다. 인증·온보딩, 상세페이지 전환·인스타 피드 모듈(스펙 미착수 — 비활성 탭만 표시)은 이번 범위에서 제외한다. **모델컷 업로드는 2026-07-22 개정으로 범위에 포함**됐다(HOME-02b). 내용 정본은 `../02-thumbnail-converter-spec.md`(파이프라인 6단계·검수 게이트), 스타일 정의는 `../../research/jp-thumbnail-style-taxonomy.md`, 프롬프트·슬롯 계약은 `data/processed/thumbnail-style-prompts.json`(v1.1.0)이다. 본 문서는 화면 동작·상태·예외처리를 다룬다.
