# 0. 개요

썸네일 스튜디오는 한국 썸네일/제품컷 1장을 입력받아, 일본 고객이 익숙한 썸네일 문법 8종 중 1개로 재설계된 일본향 썸네일을 생성하는 기능이다. 범용 이미지 생성툴이 아니다 — 일본어 카피는 jp-localizer 재설계 산출(약기법 검수 통과분)만 이미지에 들어가고, 실적 배지는 근거 없이는 생성되지 않는다(`../02-thumbnail-converter-spec.md` §0 핵심 원칙).

화면은 **스튜디오 홈(생성)**과 **생성 결과 상세** 2개로 구성한다. 스튜디오 홈이 곧 생성 퍼널이다 — 원본 업로드·플랫폼·템플릿 선택·제출을 한 화면에서 처리한다. 생성 로딩은 별도 화면이 아니라 **생성 결과 화면의 시작 상태**다 — 제출하면 결과 화면으로 이동해 생성중 상태로 시작하고, 파이프라인 단계를 거쳐 완료 시 그 자리가 결과로 crossfade 전환된다. 생성 중 다른 화면으로 이동해도 처리는 계속된다. 홈 최근 생성 스트립·③ 운영 자산 라이브러리 그리드·⓪ 메인 진행 패널은 완료를 확인하는 **보조 표면**이다. 썸네일 보관·재열람의 정본 위치는 **③ 운영 자산 라이브러리**다(`../04-operations/04-operations-ui-기획서.md` # 1 자산 라이브러리 · # 2 자산 상세) — 스튜디오 홈은 생성만 담당하고 갤러리·필터·재열람 기능을 두지 X.

화면 라벨은 전부 평문이다. 템플릿은 "클린 스튜디오 단독컷" 같은 이름으로만 표기하고 내부 스타일 ID(A~H)는 화면에 노출하지 X — ① 리포트의 라벨 정책(REPORT-14)을 계승한다.

이 화면이 성립하려면 데이터 계약 델타 2건이 필요하다(`../../08-data-flow.md` §6 개정 제안). (1) `GeneratedAsset`에 `status`(generating/done/failed)가 없다 — 결과 화면 생성중 시작 상태·홈 스트립 생성중 타일은 생성 시작 시점의 레코드(또는 잡 조회)를 전제한다. (2) 결과 상세의 재설계 해설은 파이프라인 ①~④단계 산출인데 현 계약은 `promptUsed`·`gateResult`만 저장한다 — `explanationJson`(styleReason·copySlots·krElementMap) 신설을 제안한다(RESULT-02·03).

앱 셸(사이드바 전역 내비·브랜드 스위처·KPI 위젯)은 ⓪ 메인 기획서 MAIN-01을 정본으로 따른다(2026-07-20 정합). PC 기준으로 설계하며 모바일은 반응형 대응만 한다. 인증·온보딩, 상세페이지 전환·인스타 피드 모듈(스펙 미착수 — 비활성 탭만 표시), 모델컷 업로드는 이번 범위에서 제외한다. 내용 정본은 `../02-thumbnail-converter-spec.md`(파이프라인 6단계·검수 게이트), 스타일 정의는 `../../research/jp-thumbnail-style-taxonomy.md`, 프롬프트·슬롯 계약은 `data/processed/thumbnail-style-prompts.json`(v1.1.0)이다. 본 문서는 화면 동작·상태·예외처리를 다룬다.
