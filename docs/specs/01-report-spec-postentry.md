## 0. 정합성 메모 (재논의 금지 · H5 대응)

- **확정 전제(재논의 안 함):** 빌드 확정 · 3축 구조 · Report 30만 · 증거 원칙(가상 스탯·익명 사례·가짜 리뷰 금지) · 금지 포지션(번역/종합대행/성과보장) — [[../decisions/DECISIONS]].
- **이 세그먼트의 위치:** 기존 문서의 **S2(운영중·정체)** = 본 세그먼트. [[../06-유저니즈-비즈니스-가설]] §2·H5에서 S2는 *"리뷰 메커니즘 서사만으로는 '그래서 리뷰가 쌓이냐' 불신을 못 뚫어 2차 타겟으로 강등"* 된 이력이 있다.
- **H5를 다시 뚫는 방법(본 스펙의 핵심 가설):** 서사·처방 목록이 아니라 **"교체 가능한 실물 산출물(썸네일 1 + 상세페이지 1)"** 을 30만에 포함해, 브랜드가 **자기 채널에서 개선 여부를 직접 확인**하게 한다. "그래서 되냐"에 문장이 아니라 **실물**로 답한다. → 이 실물이 구독(②③) 전환의 트라이얼이 된다(§6).
- **30만의 위상 재정의(중요):** 본 세그먼트에서 30만은 **마진 상품이 아니라 "구독 획득용 유료 파일럿(트라이얼)"** 이다. 산출물을 후하게 주는 것은 의도된 설계이며, 수익은 구독(②③)에서 회수한다(§6). 이 위상을 팀 공유 전제로 고정한다.

## 1. 목표 · 비목표 (S1 대비 델타)

### 계승 (S1과 동일)
9블록 구조 · A~E 루브릭 결정적 집계 · 약기법 전수 감사(1차 스크리닝 + 실명 검수 서명) · 코퍼스 근거 노출 · 증거 원칙 · 금지 포지션 경계. — [[01-report-spec]] §1.

### 추가·변경 (S2 전용)
1. **진단 프레이밍 전환:** "입점 준비·上市 전 점검" → **"판매 중 상세 재진단 — 왜 전환이 안 되는가."** 표지·총평·약기법 톤을 *라이브 리스크(지금 이 문구가 걸려 있음)* 로 바꾼다.
2. **퍼널 분해 진단(신규 핵심 블록):** 매출 부진의 원인을 **노출/유입(a) → 클릭(b·썸네일) → 전환(c·상세)** 으로 분해하고, **우리가 책임지는 구간은 (b)(c)** 임을 명시한다. 원인이 (a)면 *"상세를 고쳐도 한계, 유입부터"* 라고 정직하게 고지한다(§4-신규A). → "고쳤는데 안 오름 → 효과 없음 → 이탈" 리스크를 사전 차단(H5 방어).
3. **리뷰 0 대응 — 경쟁사 리뷰 마이닝:** 매출 저조 소브랜드는 **자기 리뷰가 0/희박**일 수 있다. 이때 리뷰 인과 서사(베이스 블록6)를 *자기 리뷰* 대신 **"당신이 고객을 뺏기고 있는 일본 상위 경쟁 상품의 실측 리뷰"** 인용으로 대체한다(§4-블록6 변형). 날조 없음(실제 경쟁사 리뷰) → 증거 원칙 유지.
4. **현재 실물 대조:** S1은 데모/예시 라벨이 불가피했으나, S2는 **교체할 라이브 자산(현재 상세·썸네일)이 이미 존재**한다. Before = *당신의 현재 실물* → After = *재설계 실물* 로 대조 강도를 최대화한다.
5. **실물 산출물 포함(§5):** 리포트에 더해 **① 일본향 제품 썸네일 1개(클릭 단계) + ② 상세페이지 1건(전환 단계)** 를 30만에 포함한다. 이 둘은 예시가 아니라 **당신 제품의 완성 실물**이며, **각 요소에 재설계 근거가 주석으로 붙는다**(크몽 차별화, §5.3).
6. **리뷰 축적 병목 진단(신규 핵심 · B단계 검증 반영):** 본 세그먼트의 실제 핵심 병목은 상세 카피가 아니라 **리뷰(口コミ)가 안 쌓이는 것**이다(@cosme·Qoo10 랭킹은 리뷰 수·평점이 알고리즘 입력). 리포트가 이를 정면으로 진단·설계하지 않으면 S2에서 H5가 재현된다(B단계 시뮬 P09·P11 독립 발화, §8). → **"왜 리뷰가 안 쌓이는지 진단 + 초기 리뷰 확보 설계"** 를 정식 블록으로 추가한다(§4.5). 단 **리뷰 확보의 실행·보장은 하지 않는다**(스테마 규제·성과보장 회피) — 설계까지가 리포트, 실행은 구독(③)으로 연결한다.

### 비목표 (S2에서 특히 경계)
- **유입·광고 대행 아님:** 퍼널 (a) 구간(노출·광고 집행·계정 운영)은 진단만 하고 **실행은 하지 않는다**(금지 포지션 6·종합대행). "유입이 원인"이면 그 사실을 알려줄 뿐, 유입을 우리가 만들지 않는다.
- **성과 보장 아님:** "이 상세로 바꾸면 매출 N% 상승" 금지. 산출물은 *개선 여지를 브랜드가 직접 확인할 수 있게* 할 뿐, 상승을 약속하지 않는다(§6 측정은 브랜드 자체 관측).
- **번역·저가 제작툴 아님:** 산출물은 "예쁜 일본어 상세"가 아니라 **"일본 고객 관점 재설계가 적용되고 그 근거가 증명되는 실물"** 이다. 크몽류 저가 제작(1000×1000px 1~2만원대)과 같은 링에서 싸우지 않는다(§2.3).

## 2. 사용자 · 진입

### 2.1 세그먼트 정의 (S2 하위 분기)
- **S2-a · 운영중, 리뷰 있음:** 楽天/Qoo10/@cosme에 리뷰가 쌓여 있으나 매출·랭킹 정체. → **자기 리뷰 실측**(베이스 블록6 원형)이 가능. 진단 재료 풍부.
- **S2-b · 매출 부진, 리뷰 0/희박:** 진출은 했으나 리뷰조차 안 붙음(유입·전환 동시 약함 가능). → 자기 리뷰 없음. **경쟁사 리뷰 마이닝 + 경쟁 벤치마크 + 실물 산출물**로 진단 무게중심 이동. 본 스펙이 특히 겨냥하는 난이도 높은 케이스.
- 입력 폼에서 리뷰 유무를 강제로 묻지 않고, 리뷰 URL 유무로 자동 분기한다(§3).

### 2.2 퍼널 위치
`Free(약기법 체커) → **Report 30만(=구독 트라이얼 허브)** → Growth/Scale 구독(②③)`.
S1과 달리 **Report의 목적이 "판단"이 아니라 "구독 전환"** 이다. 리포트 말미 CTA는 산출물 교체 → 개선 확인 → 구독 동선으로 설계(§6).

### 2.3 크몽 앵커 대응 (포지셔닝)
- 시장에 **일본어 상세페이지 저가 제작(크몽, 1000×1000px 1~2만원대)** 이 존재한다. 우리는 이 가격과 **정면 경쟁하지 않는다.**
- 프레임: **"우리는 크몽의 대체재가 아니라, 크몽 티어로 실패한 브랜드의 다음 단계."** 본 타깃은 이미 외주(=저가 제작)로 만들었고 매출이 저조하다. 리포트 첫 메시지 = *"저가 제작으로는 안 팔린 이유가 여기 있습니다(진단) → 이번엔 근거 있게 만든 실물(산출물)."*
- 30만이 2만과 구별되는 근거 3종(산출물에 반드시 legible하게 반영, §5.3): ① **근거가 붙은 실물**(왜 이 문장인지 = 약기법·코퍼스·구매반대이유) · ② **약기법 안전**(저가 번역은 위법 문구를 그대로 옮김; 우리는 실명 검수 + 합법 대체) · ③ **재사용 메시지 자산**(KR 병기, 이후 운영·신제품에 계속 사용).

## 3. 입력 스펙 (S1 티어 + 추가 필드)

베이스 Tier 0/1/2([[01-report-spec]] §3)를 계승하고, **입점 후 전용 필드**를 추가한다. 원칙 동일: Tier 0만으로 리포트 성립, 상위 티어는 정밀도 게이지, 가짜값 금지.

### 3.1 추가 필드
| 필드 | 티어 | 용도 | 폴백 |
|---|---|---|---|
| `liveDetailUrl` 현재 라이브 상세 URL | Tier 0 대안 입력 | 진단 대상 = **현재 판매 중 상세**. Before 실물의 소스 | URL 실패 시 텍스트 붙여넣기 1급 경로(S1과 동일) + 스크린샷 업로드(v2) |
| `salesChannel` 판매 채널 | Tier 1 | 楽天/Qoo10/Amazon JP/자사몰 — 썸네일·상세 규격, 퍼널 진단 기준 | 미상 시 Qoo10 가정 + 규격 경고 |
| `reviewUrl` 리뷰 페이지 URL | Tier 1 (분기 키) | **있으면 S2-a**(자기 리뷰 실측) · **없으면 S2-b**(경쟁사 리뷰 마이닝) | 없음=경쟁사 리뷰 마이닝으로 자동 전환 |
| `competitorUrl` 경쟁 상품 URL | Tier 2 | S2-b 경쟁사 리뷰 마이닝·벤치마크 정밀화. 비면 카테고리 상위 자동 선정 | 카테고리 코퍼스 상위로 대체 |
| `funnelSignal` 유입/전환 신호 | Tier 2 | (선택) 노출수·클릭률·전환율·체류 등 아는 값. 퍼널 분해 정밀화 | 결측 시 정성 진단(§4-신규A) |

- **성과 데이터는 "아는 것만"**: `funnelSignal`은 강제하지 않는다. 없으면 정성 퍼널 진단으로 대체하고 가짜값 금지.

## 4. 출력 — 블록 재배치 + 신규 (베이스 9블록 대비)

베이스 9블록(0 표지 ~ 9 맺음)의 **순서·무게를 재배치**하고 신규 블록 2종을 삽입한다. 블록 본문 구현 규격은 [[01-report-spec]] §4를 계승.

### 4.1 재배치 요지
- **표지(블록0):** "입점 준비" → **"판매 중 상세 재진단"** 카피. 종합점수 옆에 *퍼널 진단 요약*(어디가 새는가) 1줄 추가.
- **블록 순서:** 0 표지 → 1 요약 → **[신규A] 퍼널 분해 진단** → **[신규C] 리뷰 축적 병목 진단** → 2 페르소나·구매반대이유·USP → **6′ 경쟁사/자기 리뷰 인과** (S2-b 대응 위해 상단으로 이동) → 3 약기법(라이브 리스크 톤) → 4 경쟁 벤치마크 → 5 A~E 점수 → 7 재작성 → **[신규B/8 확장] 현재 실물 Before/After(산출물 프리뷰)** → 9 맺음(→구독 CTA).
- **정직 고지의 다음 스텝(검증 반영):** 신규A·C에서 원인이 "우리 산출물(썸네일·상세) 밖"(유입·리뷰 축적)으로 판명되면, 고객은 *"30만 내고 진단만 받았다"* 로 이탈할 위험이 있다(시뮬 P09·P11 공통 우려). → 각 정직 고지 뒤에 **구독(②③)으로 그 원인을 다루는 다음 스텝 동선**을 반드시 붙인다.

### 4.2 신규A · 퍼널 분해 진단 (신규 핵심)
- **목적:** 매출 부진의 원인이 **(a) 노출·유입 / (b) 클릭(썸네일) / (c) 전환(상세)** 중 어디인지 분해.
- **출력:** 3단 퍼널 도식 + 각 단계 진단 + **"우리가 책임지는 구간 = (b)(c)"** 명시. `funnelSignal` 있으면 실측 대입, 없으면 정성 진단(예: 리뷰 0 + 상세 약함 → 전환 이전에 클릭·유입 의심).
- **정직 고지 규칙:** 원인이 (a)로 강하게 의심되면 *"상세·썸네일을 고쳐도 한계가 있으며 유입부터 점검 필요"* 를 리포트에 명시한다(성과 보장 회피 + H5 이탈 방어). 유입 실행은 우리 범위 밖(§1 비목표).

### 4.3 블록6′ · 리뷰 인과 서사 (S2-a/S2-b 분기)
- **S2-a(리뷰 있음):** `reviewUrl` 실측 → *자기 리뷰 표현 → 구매심리 → 카피 영향* 인과(베이스 블록6 원형, 단 MVP 실측 여부는 §8-Q).
- **S2-b(리뷰 0):** **경쟁사 리뷰 마이닝** — "당신이 놓친 일본 고객이 **경쟁 상위 상품 리뷰**에 남긴 말"을 실측 인용(실제 리뷰, 날조 금지)하고, *그 고객이 확인하려던 정보를 당신 상세는 안 주고 있음* 을 연결. 표지·배너에 **"경쟁사 실측 · 자사 리뷰 미보유"** 를 명확히 라벨.

### 4.4 블록3 · 약기법 (톤 전환)
판정·조항·합법 대체·실명 서명은 베이스 계승. **톤만** "上市 전 권고" → **"현재 라이브 문장의 즉시 리스크"** (지금 걸려 있는 위법 소지 문구를 우선순위로). 소규모·리뷰 0 브랜드에게 가장 즉각적인 후킹 요소.
- **검증 반영:** "실명 검수자"만으로는 약하다 — 시뮬 P12가 *"검수자가 누군지·어떤 자격인지 안 나와 있어 진짜 전문가인지 모르겠다"* 고 지적. 검수자의 **자격·경력을 최소 수준이라도 명시**(개인정보 범위 내)해 신뢰를 실체화한다.

### 4.5 신규C · 리뷰 축적 병목 진단 (신규 핵심 · H5 대응)
- **목적:** "왜 리뷰가 안 쌓이는가"를 진단하고 **초기 리뷰 확보를 설계**한다 — 상세 재설계가 못 건드리는 S2의 진짜 병목(§1-6).
- **진단:** 리뷰 0/희박의 원인 분해(유입 자체 부족 / 리뷰 유도 장치 부재 / 구매 후 이탈 / 카테고리 초기 신뢰 공백) + 현 상세·구매 동선에서 리뷰가 유발되지 않는 지점.
- **설계(실행 아님):** 초기 리뷰 확보 경로 설계 — 샘플링→나노 인플루언서→UGC→랭킹 퍼널, 리뷰 유도 문안, **기프팅 브리핑 키트(스테마 PR 표기 가이드 포함)**. [[../06-유저니즈-비즈니스-가설]] §2-S2/H5의 "리뷰 메커니즘 산출물"을 진단 관점으로 편입.
- **경계(필수):** **리뷰 수·평점을 보장하지 않는다**(스테마 규제·성과보장 금지). 리포트는 *진단 + 설계*까지. 실행(지속 운영)은 **구독(③)으로 명시 연결** → 리뷰 병목이 곧 구독 전환의 근거가 된다.
- **정직 고지:** 원인이 "유입 자체 부족"이면 상세·리뷰 설계로도 한계임을 고지(§4.1 다음 스텝 동선과 연동).

## 5. 산출물 스펙 (30만 포함 — 리포트 + 실물 2종)

### 5.1 구성
| 산출물 | 퍼널 구간 | 정의 |
|---|---|---|
| 진단 리포트 9블록(재배치) | 진단 | §4. 근거·규정·검수 포함 |
| **일본향 제품 썸네일 1개** | 클릭(b) | `salesChannel` 규격 준수. 검색결과 퍼스트뷰용 완성 실물 |
| **상세페이지 1건** | 전환(c) | 재작성·USP·페르소나가 적용된 완성 상세. 채널 규격 |

> 참고: 이전 논의에서 "퍼스트뷰 1스크린만 포함 vs full 상세 포함"을 저울질했으나, 리뷰 0 세그먼트는 진단 재료가 얇아 **실물이 가치의 무게중심**이 되어야 하고, 저가 제작 앵커(크몽) 대비 "돈값 실물"이 필요하므로 **full 상세 1건 포함(B안)** 으로 확정(사용자 결정 2026-07-08). 대신 §5.2·§6 조건으로 마진·카니발 위험을 감싼다.

### 5.2 생산 구조 — 반자동 조립 (마진 방어, 필수)
- 상세·썸네일을 **0에서 수작업 제작하지 않는다.** 진단 파이프라인이 이미 뽑는 산출물(재작성 카드·USP·페르소나·합법 대체 표현)을 **designer가 조립**하는 구조로 한다.
- 즉 **AI 진단 → 반자동 상세/썸네일 조립.** 건당 사람 한계비용을 낮춰 30만 트라이얼의 지속가능성을 확보한다. (CLAUDE.md "우리도 도구를 만든다"와 정합 — 범용 생성기가 아니라 진단 내장 조립기.)

### 5.3 크몽 차별화 — "근거가 붙은 실물" (필수 요건)
산출물은 결과물만 던지지 않는다. **각 핵심 요소에 재설계 근거 주석**을 붙인다(예: 이 헤드라인=K3 회피 + 코퍼스 「整える」 프레임 + 구매반대이유① 대응). 이 주석이 리포트와 실물을 **한 몸**으로 묶어 30만을 정당화한다. 근거 없는 실물은 크몽 2만과 구별 불가 → 반드시 legible하게.
- 실물 내 수치·인증·무첨가 라벨은 **확인된 경우에만** 기입(증거 원칙). 미확인은 플레이스홀더 표기.
- **판매/랜딩 화면 요건(검증 반영 · 필수):** "재설계"를 **말이 아니라 실물 Before/After 샘플**로 보여준다. 시뮬 3명 공통 — *"말로만 재설계라고 하면 크몽 프리랜서도 똑같이 말한다; 실제 전후 샘플(+가능하면 결과)을 눈으로 봐야 15배 가격이 믿긴다."* 가격 옆에 샘플 실물을 노출한다.
- **가상 스탯 금지(검증 반영):** 랜딩의 가상 성과 수치(예: "87%", "+214%", "체류 +41%·CVR +38%·프로필 2.3배 β 12개 브랜드")는 **역효과**다 — 시뮬 P12·P11이 *"가상 데이터 같아 오히려 의심된다 / 제 얘기가 아니다"* 로 지적(증거 원칙과도 배치). 제거하고 실측·실물 샘플로 대체. → [[../../persona-simulation/landing/index.html]] 수정 액션.
- **AI 생성 이미지 경계(조사 반영 · 필수):** 산출물의 이미지는 **"구성·카피·톤을 재설계한 제안(데모)"**이며 **실판매용 제품 본체 컷은 브랜드 실촬영으로 교체** 전제다. 근거: Qoo10 메인 이미지는 실제 제품 사진 원칙(AI 렌더 제품컷은 오인·반려 리스크), 薬機法·景表法상 AI 이미지도 실사와 동일 기준(효능·전후·가짜 실증 금지, 실사 오인 시 "イメージ" 표기). 배경·연출만 AI 활용 가능. 상세: [[../research/AI생성이미지-일본플랫폼-법규-2026-07-08]]. → 시뮬 P09·P11이 이 경계 정리를 *"오히려 신뢰"* 로 수용.

## 6. 상품·가격 위상 · 구독 전환 루프

### 6.1 30만 = 트라이얼(획득 가격)
마진 상품이 아니라 구독(②③) 획득용 유료 파일럿. 산출물을 후하게 주는 것이 설계 의도(§0).

### 6.2 스튜디오②의 재정의 (카니발 방지)
30만에 상세 1건이 들어가면 ②가 "1건 제작"이면 충돌한다. → **② = "1건"이 아니라 "지속·볼륨"**: 신제품마다·시즌마다(메가와리 연4회 등)·A/B 개선·복수 채널 전개. 30만은 **증명용 1건**, 구독은 **흐름**을 판다.

### 6.3 개선 확인 → 구독 루프
1. 산출물(썸네일·상세) 교체 → 2. **브랜드가 자기 채널에서 개선 여부 직접 관측**(클릭률·전환율·저장·리뷰 유입) → 3. 개선 여지 확인 → 4. 구독(②③) 전환.
- **측정은 브랜드 자체 관측**이며 우리가 성과를 보장·대리측정하지 않는다(성과보장 금지). 리포트는 *"무엇을 보면 개선을 확인할 수 있는지"*(관측 지표 가이드)까지만 제공.
- **관측 지표 명시(검증 반영):** 시뮬 3명이 구독 전환 조건으로 *"개선을 숫자로 보여줄 것"* 을 공통 요구 — **썸네일 교체 전후 CTR, 상세 교체 전후 전환율, 리뷰 유입 수**를 관측 지표로 리포트에 명시.
- **구독 가격 사전 공개(검증 반영 · 필수):** 시뮬 3명 공통 불안 = *"월 구독이 얼마인지 안 나와 있다; 나중에 훅 올려부를까 봐"*. → 30만 제안 시점에 **구독 가격대(밴드)와 월 산출물 범위를 미리 명시**한다. 불투명하면 구독 전환이 막힌다.
- (a·유입 원인) 케이스는 이 루프가 깨지므로 §4.2 정직 고지로 **애초에 (b)(c) 원인 브랜드만 남긴다.**

### 6.4 확정 가격·구성 (2026-07-08~09) + 3차 검증 발견
- **확정:** 진단 리포트+썸네일+상세 = **30만/1회** · 구독 **월 20만원** · **기프팅·인플루언서 실비 별도**(브랜드 부담, 우리는 설계·브리핑·운영).
- 🔴 **핵심 발견(3차 재시뮬):** 구독 전환 병목이 "가격"에서 **"플랜 구성"**으로 이동. 20만 구독 플랜에 **"리뷰 축적 운영"이 명시적으로 없으면** S2 페르소나(핵심 통증=리뷰)는 사인하지 않는다(P11: *"Growth 항목에 '리뷰'가 없다, 리뷰·랭킹은 Scale 80만에만 있다"*). → **리뷰 축적 운영을 20만 구독의 명시 항목으로 편입**해야 한다(현 랜딩 요금표는 S2 전략과 불일치 — 리뷰가 상위 티어에 가 있음).
- 🟡 **설계→실행 브릿지(P11):** 구독 초기 **"첫 4주 후 리뷰/CTR 전후 확인 지점"**을 플랜에 명시하면 사인 장벽이 내려간다. 30만(설계)→20만 구독(실행) 사이 신뢰 공백을 이 조기 성과 확인점이 메운다.
- 🟡 **구독 산출물 주기 명시(P09):** "신제품·시즌마다"는 신제품 뜸한 소규모(8인)엔 *"월 20만 내는데 뭐 받는지"* 불명 → **월 최소 산출물(콘텐츠 N건 등)**을 명시.
- 🟡 **실비 가이드(P09):** 별도는 수용되나 *"대략 얼마"*(예: 마이크로 인플루언서 1인당) 개략 가이드 있으면 좋음.
- ✅ **정상 퍼널 확인(P09):** "리포트 받아보고 만족하면 구독"이 자연스러운 순서 → 구독 "조건부"는 이탈이 아니라 **의도된 리포트→구독 전환 동선**.
- ✅ **4차 재시뮬(2026-07-09) — 구독 병목 돌파:** 리뷰 운영 편입 후 **P11 "간다(사인)", P09 "리포트 후 거의 간다"**. S2 전환 루프 검증 완료([[../research/simulations/시뮬-postentry-재시뮬-2026-07-08]] §4차).
- ✅ **"키트 vs 실행 대행" 경계 확정(2026-07-09):** **20만 Growth = self-serve 키트**(매칭 리스트·컨택 템플릿·SOP·브리핑). 실제 컨택·협상·기프팅 발송은 브랜드가 하거나 **"컨택 대행" 유료 add-on**으로 선택. → 20만 마진 보존 + 리소스 부재 S2도 포섭. (실비는 여전히 별도.)
- ✅ **부차 폴리시 기본값(요금표 반영):** 콘텐츠 **월 15~30건(채널·시즌에 따라)** · 나노 인플루언서 **후보 리스트 월 10명 내외** · 실비 **1인당 10만원대 · 초기 월 3~5명 권장** · 점검 **첫 4주 성과 확인 → 이후 월 1회 라이트 체크 + 분기 심층 재점검**(리뷰 초조 상황의 "중간 2달 공백" 해소).

## 7. 증거 원칙 · 금지 포지션 재확인 (S2 특화)

## 8. B단계 검증 결과 (페르소나 시뮬 2026-07-08)

대상: P09(운영 6개월 정체)·P11(3개월·리뷰0)·P12(외주 데인·색조) + 대조군 P19(미진출). 자극물 = 고객용 30만 패키지 제안.

### 8.1 결과 요약
- **Q1 지불의향:** 세그먼트 내 3명 **전원 "조건부"**, 즉시 "낸다" 0명. 30만 액수 자체는 부담 아님(외주 1회분). 병목은 **확신 부재**.
- **Q2 구독 전환:** 3명 **조건부 예스** — 조건 2개 공통: ① 개선을 **숫자로** 보여줄 것 ② 구독 **가격 사전 명시**. (→ §6.3 반영)
- **Q3 정직 고지:** **4명 전원 신뢰 상승** ✅ ("안 되는 걸 안 된다고 하네"). 단 그림자: *"원인이 산출물 밖이면 30만 내고 진단만?"* → 다음 스텝 동선 필요(§4.1 반영).
- **세그먼트 프레이밍:** 대조군 P19가 **"나는 대상 아니다"로 명확히 판별** ✅ — 입점 후 타깃팅 정상 작동.

### 8.2 반영 완료(본 개정)
- 🔴 **리뷰 축적 병목 = H5 재현** (P09·P11 독립 발화): "내 문제는 상세가 아니라 리뷰가 안 쌓이는 것; 경쟁사 리뷰 분석은 내 리뷰를 쌓는 게 아님." → **신규C 블록 추가**(§4.5), §1-6.
- 🟡 **크몽 차별화 legible화:** 실물 Before/After 샘플 + 가상 스탯 제거 → §5.3.
- 🟡 **구독 투명성:** 관측 지표 + 가격 사전 공개 → §6.3.
