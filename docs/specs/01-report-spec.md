## 1. 목표 · 비목표

### 하는 것 (목표)
1. 한국 뷰티 브랜드의 **KR 상세페이지 콘텐츠**를 입력받아, 일본 시장 관례(라쿠텐 코퍼스 근거) 대비 **무엇이 부족한지**를 A~E 루브릭으로 채점한 **일본 문법 진단 점수**를 산출한다.
2. 일본 고객 관점의 **타깃 페르소나**와 **USP 재정의**(한국식 소구 → 일본 구매이유)를 제시한다.
3. 저점 항목을 근거로 **NG/OK 재작성(Before/After)** 문장쌍과 **비포&애프터 샘플 1종**을 제시한다.
4. 모든 산출에 **채점 기준·코퍼스 근거를 함께 노출**해 "번역이 아닌 재설계"를 측정 가능하게 만든다(신뢰 장치).

### 안 하는 것 (비목표 — 금지 포지션 경계)
- **번역 서비스 아님**: 문장을 일본어로 옮기지 않는다. 채점·재작성은 "정보 구조(근거 라벨·등급·무첨가·성분 정량·서사)"를 진단·재설계하며, 번역 품질을 매기지 않는다(루브릭 §4 말미 원칙). → 금지 포지션 5(번역).
- **종합 대행 아님**: 물류·통관·인허가·광고 집행·계정 운영을 하지 않는다. 리포트는 **사전 진단 문서**에서 끝난다. → 금지 포지션 6(종합 대행).
- **약기법 법률 판정 아님(MVP)**: 위반 여부의 법적 확정 검출은 Free 체커(별도 모듈)·v2 영역. 리포트의 A3(의약외품 등급)은 "표현이 등급에 맞는가"를 **재설계 관점 힌트**로만 다루고, 법적 위반 단정은 하지 않는다(§9-Q4).
- **가상 성과 약속 아님**: "매출 N% 상승" 같은 예측·보장 문구 금지. 벤치마크 수치는 **코퍼스 실측치만** 사용(증거 원칙).

## 2. 사용자 · 진입 (퍼널 위치)

- **1차 타깃 페르소나: S1(입점 준비/직전 브랜드)** — [[../03-personas]] · [[../06-유저니즈-비즈니스-가설]] §2.
  - 맥락: "국내 매출은 있으나 일본은 처음. 대행사에 큰돈 쓰기 전에 내 콘텐츠가 통할지, 뭘 고칠지 알고 싶다."
  - 구매 트리거: 추상 슬로건이 아니라 **구체적 "고쳐야 할 문구 예시"**(Before/After)로 전문성 체감.
- **보조 타깃: S4(규제 민감)** — A3·NG/OK 재작성에서 부분적으로 소구. 단 상시 체크 니즈는 Free 체커/v2 담당.
- **퍼널 위치:** `Free(약기법 간이 체커·체크리스트) → **Report 30만(본 리포트=전환 허브)** → Growth → Scale`.
  - Free 체커는 "위반 표현 검출"까지. **재작성·페르소나·USP·A~E 문법 점수는 Report에서만** 제공(무료/유료 경계 — [[../06-유저니즈-비즈니스-가설]] §6).
  - 화면 흐름 함의: Free 체커 결과 화면에 "전체 문법 진단 + 재작성 = Report" 업셀 지점을 둔다(designer 연계).

## 3. 입력 스펙

결정 문서 "소비자 제공 정보" 표를 **MVP용으로 취사선택**한다. 원칙: **최소 입력(카테고리 + 콘텐츠 텍스트)만으로도 리포트가 나온다.** 입력이 많을수록 정밀해진다.

### 3.1 입력 필드

| 필드 | 필수/선택 | 타입 | 검증 규칙 | 없을 때 폴백 |
|---|---|---|---|---|
| `brandName` 브랜드명 | 선택 | text | 1~60자 | "(브랜드명 미기재)"로 표기, 페르소나·USP는 콘텐츠에서 추론 |
| `productName` 제품명 | 선택 | text | 1~120자 | 콘텐츠에서 추출 시도 |
| `category` 카테고리 | **필수** | enum | `skincare`/`makeup`/`suncare`/`cleansing` 중 1 | — (필수. E군 가중·카테고리 코드 채점의 키) |
| `productClass` 제품 분류 | **필수** | enum | `화장품`/`의약외품(의약부외품)` | 미상 시 `화장품`으로 가정하고 리포트에 "분류 미확인 — 등급별 허용 표현 상이" 경고 표기 |
| `sourceType` 입력 방식 | **필수** | enum | `url`/`text` | — |
| `sourceUrl` KR 상세페이지 URL | 조건부 | url | `sourceType=url`일 때 필수. http(s), 도달 가능 | 스크래핑 실패 시 `text` 입력으로 폴백 유도(에러 메시지에 대체 경로 안내) |
| `sourceText` 콘텐츠 텍스트 | 조건부 | textarea | `sourceType=text`일 때 필수. 최소 200자 권장(최소 50자 미만이면 경고) | 200자 미만이면 "진단 신뢰도 낮음" 배지 |
| `keyIngredients` 핵심 성분 | 선택 | text[] | 각 1~40자, 최대 8개 | D군 채점은 콘텐츠 내 성분 표현 기준으로만 |
| `priceJPY` 예상 판매가(엔) | 선택 | number | ≥0 | 페르소나 가격 감도 추론에서 제외 |
| `instagramHandle` 인스타 계정 | 선택 | text | `@`+영숫자 | (MVP는 미수집·미분석. v2 KPI 재정의용) |
| `targetMemo` 타깃/고민 메모 | 선택 | textarea | 0~500자 | 페르소나 생성 보조 입력 |

### 3.2 최소 입력 폴백 규칙 (반드시 구현)
- **최소 성립 조건:** `category` + `productClass` + (`sourceUrl` 스크래핑 성공 텍스트 OR `sourceText` 50자+). 이 조건만 충족하면 리포트를 생성한다.
- 선택 필드 결측 시 **해당 근거의 채점·생성을 건너뛰거나 콘텐츠 추론으로 대체**하되, 리포트에 "이 입력이 있으면 더 정밀해집니다" 안내를 표기(가짜로 채우지 않음 — 증거 원칙).
- **KR 상세는 이미지 위주가 많다**는 현실 리스크: `url` 실패·텍스트 빈약 시 `text` 붙여넣기를 1급 경로로 안내한다(§9-Q1, 이미지 OCR은 v2).

## 4. 리포트 산출물 구조 (섹션 정의)

리포트는 아래 순서로 렌더링한다. 각 섹션은 **근거 노출**을 동반한다(증거 원칙).

### (헤더) 진단 요약
- 브랜드/제품/카테고리/분류, 진단 일시, **일본 문법 진단 종합 점수(0~100)** + 3~4줄 총평, 최우선 재설계 3가지(저점 항목 top 3).
- 신뢰 배지: "채점 기준 공개" · "라쿠텐 상세 304건 코퍼스 기준" · (있으면) "샘플 리포트 대비".

### (a) 일본향 타깃 페르소나
- 산출: 1인 페르소나 카드 — 이름/연령대·피부고민·구매 동기·구매 전 확인 행동(예: `@cosme·LIPS 리뷰 확인`)·가격 감도·신뢰 트리거.
- **일본 구매 여정 반영**: 인스타=인지/저장 → 리뷰(口コミ) 검증 → 전환 구조([[../06-유저니즈-비즈니스-가설]] §3, 리서치 §10). "이 페르소나는 어디서 최종 확신을 얻는가"를 명시.
- 근거 노출: 카테고리 코퍼스 패턴(예: makeup→퍼스널컬러 프레임) 인용.

### (b) USP 재정의 (한국식 소구 → 일본 구매이유)
- 산출: 표 형식 — `현재(KR) 소구` → `일본 고객에게 읽히는 방식(리스크)` → `재정의된 일본향 구매이유(USP)`. 3~5행.
- 원칙: 한국식 "즉각 효과·감성 형용사"를 **일본식 "근거·등급·무첨가·성분 정량·제3자 지표"** 로 전환(리서치 §3). **번역이 아니라 구매이유 재설계**임을 문장으로 못박음.

### (c) 일본 문법 진단 점수 (A~E 루브릭 — 핵심 엔진)
- **채점 원본:** `research/jp-detail-message-patterns.md` §4. 항목 A1~A5·B1~B2·C1~C3·D1~D4·E1~E4를 **각 0(없음)/1(부분)/2(관례 충족)** 로 채점.
- **표현 방식(채점기준 공개 원칙):** 각 항목마다 `점수 · 통과기준 · 판정 근거(고객 콘텐츠의 실제 문장 인용) · 코퍼스 근거(관례 예시)` 4요소를 노출한다. 점수만 던지지 않는다.
- **집계·가중 (MVP 확정 공식):**
  - 그룹 점수(%) = (그룹 내 항목 점수 합) ÷ (적용 항목 수 × 2) × 100.
  - **E군은 제품 카테고리에 해당하는 항목만 채점**(suncare→E1, makeup→E2, cleansing→E3, skincare→E4). 나머지 E항목은 N/A로 제외(분모에서도 제외).
  - **종합 점수** = Σ(그룹 점수 × 그룹 가중치). 가중치는 카테고리별로 다르다(아래 표, 합=1.00).

  | 그룹 | skincare | suncare | makeup | cleansing |
  |---|---|---|---|---|
  | A 신뢰 구축 | 0.30 | 0.20 | 0.15 | 0.20 |
  | B 무첨가·안전 | 0.15 | 0.10 | 0.10 | 0.30 |
  | C 서사 구조 | 0.25 | 0.10 | 0.20 | 0.15 |
  | D 성분 프레이밍 | 0.20 | 0.20 | 0.15 | 0.15 |
  | E 카테고리 적합성 | 0.10 | 0.40 | 0.40 | 0.20 |

  > 가중치는 루브릭 §2-d·판정 가이드(A·B·D 저점=최우선, 카테고리별 지배 문법)에 근거한 **초기값**. 파일럿 튜닝 대상(§9-Q2).
- **표시:** 종합 점수 + 그룹별 막대 + 항목별 상태(충족/부분/미흡). 종합 점수는 **"일본 상세 관례 충족도"** 로 프레이밍(절대 등급·성과 예측 아님). 밴드 라벨(재설계 시급/보완 권장/양호)의 임계값은 §9-Q3.

### (d) NG/OK 재작성 (Before/After 문장쌍)
- 산출: **저점 항목(0~1점)에서 도출된** 고객 콘텐츠의 문제 문장 → 일본향 재설계 예시. 3~5쌍(MVP).
- 각 쌍: `Before(고객 원문)` · `문제(어느 루브릭 항목 위반·왜)` · `After(재설계안)` · `무엇을 더했나(근거 라벨/각주/무첨가/성분 정량/서사)`.
- After는 일본어 표현을 포함할 수 있으나 **목적은 정보 구조 재설계**임을 명시(번역 산출물이 아님). 실측 코퍼스 표현·렉시콘 어휘를 근거로만 사용, 가짜 수치·가짜 인증 삽입 금지.

### (e) 비포&애프터 샘플 1종
- 산출: 문장 단위가 아닌 **한 블록(예: 상단 히어로 카피 or 성분 소구 블록) 통째 재구성** 1종. 리포트 말미의 하이라이트 사례.
- MVP: 고객 콘텐츠 중 **가장 저점인 섹션**을 골라 재구성. 고객 콘텐츠가 빈약하면 **동일 카테고리 대표 패턴 기반 데모 샘플**로 대체하되 "예시(데모)" 라벨을 명확히 표기(실물 브랜드 사칭·익명 사례 금지).

## 5. 산출 로직 (개발자용)

각 섹션을 무엇으로 생성하는가 — **규칙기반(루브릭·정규식) vs Claude 생성**을 분리한다.

### 5.1 파이프라인 개요
```
입력 → [1 콘텐츠 수집/정규화] → [2 사전 신호 추출(규칙)]
     → [3 루브릭 채점(LLM, 구조화 출력)] → [4 집계·가중(규칙)]
     → [5 페르소나·USP 생성(LLM)] → [6 NG/OK 재작성 + 샘플(LLM)]
     → [7 리포트 조립·렌더]
```

### 5.2 단계별 정의

| 단계 | 방식 | 입력 | 출력 | 코퍼스/렉시콘 grounding |
|---|---|---|---|---|
| 1. 콘텐츠 수집·정규화 | 규칙(파서) | `sourceUrl`(fetch→본문 텍스트) 또는 `sourceText` | 정규화 평문(줄바꿈 정리, 중복 제거) | — |
| 2. 사전 신호 추출 | 규칙(정규식) | 정규화 평문 + `keyIngredients` | 신호 플래그: 숫자·`%`·SPF/PA·"임상/시험/인증"·"무향/무첨가"·"랭킹/1위/리뷰수"·성분 후보 | — |
| 3. 루브릭 채점 | **LLM 1콜(구조화 JSON)** | 정규화 평문 + 사전 신호 + `category`/`productClass` + **루브릭 §4 전문** + **코퍼스 벤치마크 요약** | 항목별 `{score:0/1/2, evidenceQuote, criterionRef, corpusRef}` | **주입:** 루브릭 항목·통과기준, 코퍼스 상위 배지/성분/카테고리 패턴(`detail-ocr.jsonl` 집계 요약), 렉시콘 상위 어휘(`sns-lexicon.csv`) |
| 4. 집계·가중 | 규칙(순수 함수) | 단계3 점수 + 가중표(§4-c) | 그룹 점수·종합 점수·저점 top3 | — |
| 5. 페르소나·USP | **LLM 1콜** | 콘텐츠 요약 + `category`/`priceJPY`/`targetMemo` + 카테고리 코퍼스 패턴 + 페르소나 여정 원칙 | 페르소나 카드 + USP 재정의 표(JSON) | 카테고리 관례(makeup=퍼스널컬러 등), 구매 여정(인스타→리뷰→전환) |
| 6. NG/OK 재작성 + 샘플 | **LLM 1콜** | 저점 항목(단계3·4) + 해당 문제 문장 + 코퍼스 재설계 예시 + 렉시콘 | NG/OK 3~5쌍 + 비포&애프터 샘플 1종(JSON) | 코퍼스 실측 표현(`効能評価試験済み`·`○○フリー`·`6%`·랭킹 표기), 렉시콘 실제 JP 어휘 |
| 7. 조립·렌더 | 규칙 | 3~6 산출 | 리포트(웹뷰 + 내보내기) | — |

- **LLM 콜 수(MVP):** 3콜 목표(채점 / 페르소나·USP / 재작성·샘플). 라운드트립·비용 절감을 위해 5·6은 콘텐츠 요약을 공유. 필요 시 4콜까지 허용.
- **구조화 출력:** 모든 LLM 콜은 스키마 고정 JSON으로 받는다(파싱 안정성). 프롬프트는 "코퍼스 근거 밖의 수치·인증을 창작하지 말 것"을 명시(증거 원칙 가드레일).
- **grounding 주입 형태:** 코퍼스·렉시콘 원본을 통째로 넣지 않고 **사전 집계된 벤치마크 요약**(상위 배지·성분·카테고리 문법 표 — 리서치 §1·§2에서 이미 정리됨)을 프롬프트 컨텍스트로 주입. 대용량 원본은 v2에서 검색형(RAG)으로.
- 구현 규칙(camelCase·JSDoc·로거·타입 안전·`any` 지양·접근성)은 **구현 시 CLAUDE.md 준수**. 본 스펙은 로직 계약만 정의.

### 5.3 규칙기반 vs LLM 경계 (핵심)
- **규칙기반:** 집계·가중(4), 사전 신호 정규식(2), 콘텐츠 파싱(1) — 재현 가능·검증 가능해야 하는 부분. **점수 계산은 결정적(deterministic)** 이어야 한다(같은 항목 점수 → 같은 종합 점수).
- **LLM:** 항목 0/1/2 판정(3), 페르소나·USP·재작성(5·6) — 판단·생성이 필요한 부분. LLM은 **점수를 매기되 가중·합산은 하지 않는다**(4는 코드가 담당). 이로써 채점 기준 공개·재현성을 보장.

## 6. 유저 스토리 + 수용 기준(AC)

### US-1 (S1) 최소 입력으로 진단
"일본 진출을 준비하는 브랜드 담당자로서, 상세페이지 텍스트를 붙여넣기만 해도 진단 리포트를 받고 싶다. 왜냐하면 대행사 견적 전에 빠르게 통할지 확인하고 싶기 때문."
- **AC-1.1** `category`+`productClass`+`sourceText`(50자+)만으로 리포트가 끝까지 생성된다.
- **AC-1.2** 선택 필드 결측 시 가짜값을 넣지 않고, 결측 안내와 함께 가능한 섹션까지 산출한다.
- **AC-1.3** `sourceUrl` 스크래핑 실패 시 텍스트 붙여넣기 폴백 경로를 에러 메시지로 안내한다.

### US-2 (S1) 문법 점수를 근거와 함께 확인
"담당자로서, 점수뿐 아니라 왜 그 점수인지(내 문장 어디가·일본 관례는 어떤지) 보고 싶다. 왜냐하면 근거 없는 AI 점수는 못 믿기 때문."
- **AC-2.1** A~E 전 항목이 0/1/2로 채점되고, 각 항목에 `통과기준·내 문장 인용·코퍼스 근거`가 노출된다.
- **AC-2.2** 종합 점수는 §4-c 가중 공식으로 계산되며, **같은 항목 점수 입력 → 같은 종합 점수**(결정적)임을 단위 테스트로 확인한다.
- **AC-2.3** E군은 제품 카테고리에 해당하는 항목만 채점·분모 반영된다.
- **AC-2.4** 리포트 어디에도 코퍼스 실측 밖의 가상 통계·익명 성공 사례·성과 보장 문구가 없다(증거 원칙 — QA 체크).

### US-3 (S1/S4) 고칠 문구 예시를 받는다
"담당자로서, 저점 항목에 대해 '이 문장을 이렇게 바꾸라'는 Before/After를 받고 싶다. 왜냐하면 구체적 예시라야 전문성을 신뢰하고 실행할 수 있기 때문."
- **AC-3.1** NG/OK 3쌍 이상이 산출되고, 각 쌍이 특정 루브릭 항목 위반과 연결된다.
- **AC-3.2** 비포&애프터 샘플 1종이 말미에 포함된다. 고객 콘텐츠 빈약 시 데모 샘플은 "예시(데모)" 라벨이 붙는다.
- **AC-3.3** After는 근거 라벨·각주·무첨가·성분 정량 중 무엇을 더했는지 명시한다.

### US-4 (사업) 번역·대행으로 오인되지 않는다
"기획자로서, 리포트가 번역물/대행 제안서로 읽히지 않기를 바란다. 왜냐하면 금지 포지션과 충돌하면 차별점이 무너지기 때문."
- **AC-4.1** 리포트 문구에 "번역해드립니다/운영 대행/광고 집행" 류 표현이 없다.
- **AC-4.2** USP·재작성 섹션이 "정보 구조 재설계"로 프레이밍된다(번역 결과물 아님).

### 성공 지표 (검증 가능)
- **완주율:** UT(8/1~3)에서 테스트 사용자의 리포트 완주(입력→리포트 열람) ≥ 80%.
- **신뢰 지표:** 리포트 열람자 중 "일본에서 통할 판단에 도움됨" ≥ 4/5(정성) — [[../02-product-spec]] UT 기준 연동.
- **결정성:** 동일 입력 재실행 시 종합 점수 편차 0(집계 결정적), 채점(LLM) 재현 편차는 관찰·기록.
- **증거 원칙 위반 0건:** QA에서 가상 스탯·익명 사례·성과 보장 문구 검출 0.

## 9. 열린 질문 (→ [[../05-open-questions]] 스타일)

- **Q1. KR 상세 이미지 문제:** 한국 상세도 이미지 위주가 많다. MVP는 텍스트 붙여넣기로 회피하나, 입력 UX 이탈 우려. URL fetch가 이미지 alt/캡션만 긁을 때의 최소 품질 기준은? (이미지 OCR은 v2 확정)
- **Q2. 카테고리 가중치 수치:** §4-c 가중표는 루브릭 근거의 초기값. 실제 KR 고객 상세 5~10건 파일럿(리서치 부록 과제)으로 튜닝해야 한다. 튜닝 전 이 값으로 고정 진행 OK?
- **Q3. 종합 점수 밴드 라벨·임계값:** 0~100을 "재설계 시급/보완 권장/양호"로 나눌 임계값 미정. 표본이 적어 근거가 약함 → MVP는 밴드 없이 숫자+총평만 낼지 결정 필요.
- **Q4. A3(의약외품 등급)과 Free 약기법 체커의 관계:** 리포트 A3은 "등급 적합 힌트"만, 법적 위반 판정은 Free/별도 모듈. 이 분리가 사용자에게 혼란스럽지 않게 어디까지 노출할지.
- **Q5. 채점(LLM)의 재현성:** LLM 판정은 실행마다 미세 변동 가능. temperature·프롬프트 고정으로 어느 편차까지 허용할지, 편차를 사용자에게 고지할지.
- **Q6. 비포&애프터 데모 샘플:** 고객 콘텐츠 빈약 시 데모 대체를 허용했으나, 증거 원칙(익명 사례 금지)과의 경계 확인 필요 — "예시(데모)" 라벨만으로 충분한지.
- **Q7. 리포트 내보내기 형식:** PDF vs 웹 공유 링크 vs 인쇄뷰 — designer·frontend와 확정(신뢰 장치 "샘플 리포트 실물"과 연결).
