# 0. 개요

진단 리포트는 한국 뷰티 브랜드의 정체성(브랜드명·포지셔닝·카테고리)을 필수로 입력받아 일본 고객 관점의 페르소나·USP 재설계를 산출하는 기능이다. 상세페이지 카피는 선택 입력이며, 넣으면 薬機法 전수 감사·A~E 문법 점수·재작성이 열린다. 산출물은 진단 모드 2종으로 갈린다 — 브랜드 진단(콘텐츠 없음: 블록 1·3·5·7 데이터 잠금)과 브랜드+제품 진단(8블록 전부). 산출물은 번역이 아니라 일본 고객 관점의 메시지 재설계다.

두 모드 모두 유료 상품이다 — 브랜드+제품 진단은 고정가 30만 원, 브랜드 진단 가격은 (미정). 무료는 샘플뿐이다 — 제출 즉시 표지 전체·요약 일부(종합점수 가림)·페르소나 카드 맛보기가 열람되고, 풀 리포트는 결제 후 열람한다(두 모드 공통). 결제 잠금의 집행은 미구현이며 이번 빌드는 발행 리포트를 전체 열람한다 — 데이터 잠금(콘텐츠 미제출)만 구현 범위다.

화면은 입력 폼 → 처리 로딩 → 리포트 열람 → 보고용 슬라이드 4개로 구성한다. 앱 셸(사이드바 전역 내비·브랜드 스위처·KPI 위젯)은 ⓪ 메인 기획서 MAIN-01을 정본으로 따른다(2026-07-20 정합). PC 기준으로 설계하며 모바일은 반응형 대응만 한다. 인증·온보딩은 이번 범위에서 제외하고, 입력은 매번 폼에서 직접 받는다. 파이프라인이 성공하면 곧바로 발행되며 사람이 개입하는 단계는 없다.

리포트 열람은 **3장 서사**로 낸다(2026-07-18) — 표지 뒤에 **1장 시장**(시장 브리핑: "일본은 이렇게 판다") → **2장 진단**(블록 1 → 4 → 2 → 3 → 5 → 6: "내 브랜드는 지금 여기") → **3장 처방**(블록 7 + 해결 칩: "이렇게 바꾸면 풀린다") → 맺음(블록 8 → 닫는 밴드). 브랜드 맞춤인 블록 4 벤치마크·블록 2 페르소나는 점수(블록 1) 뒤 2장에 둔다(2차 정리) — 1장은 브랜드 무관 시장 일반만. 블록 **번호**(blocksJson ID)·채점 산식·LLM 콜 계약은 그대로이고 화면 **표시 순서·라벨**만 재배열한다 — 순서 정본은 `REPORT-14`. 화면 라벨은 전부 평문이며 A~E·K 코드는 내부 계약 ID로만 존재한다(리포트 화면 비노출).

내용 정본은 `docs/specs/01-report-spec.md`(§3 v4 입력·8블록 정의·채점 공식), 데이터 계약은 `docs/08-data-flow.md`다. 본 문서는 화면 동작·상태·예외처리를 다룬다.
