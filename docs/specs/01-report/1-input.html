1. 진단 입력

/app/report/new · 기획서 INPUT-00 ~ INPUT-05. 진단의 주어는 브랜드다. 브랜드 3종(브랜드명·포지셔닝·카테고리)만으로 리포트가 끝까지 나오고, 상세페이지 콘텐츠는 선택이다. 넣으면 약기법 감사·문법 점수·재작성이 열린다(두 진단 모드).

진입 시 GET /api/report로 저장 종류·LLM 모드를 조회해 배지로 알린다. 운영 환경에서는 둘 다 나오지 않는다. 조회가 실패해도 폼은 정상 동작한다.

제품 정보 · 선택

3a~3e · 정밀도 게이지

전부 선택 입력이다. 입력된 사실만 스냅샷에 담고, 미입력에 에러를 내지 않는다(증거 원칙 — 사실만). 콘텐츠(3e)를 넣으면 브랜드+제품 진단, 비우면 브랜드 진단이다.

제품 분류에 필수 표시(*)가 없다. 미선택은 서버가 “미상”으로 정규화한다. 분류는 약기법 감사의 판정 등급을 정하는 키인데(블록 3-0), 감사 자체가 콘텐츠가 있을 때만 돌기 때문이다. 리뷰 페이지 URL 필드는 이번 개정에서 뺐다(소비처 0 · 데드필드 · INPUT-05).

INPUT-03·04

콘텐츠 게이트 · 진단 모드

3e · 비움=브랜드 / 50자 하드 / 200자 소프트

글자수는 공백을 제외하고 센다(단일 정본 gates.ts). 게이트는 “넣다 만” 입력에만 발동한다. 0자는 잠금이 아니라 브랜드 진단이고, 1~49자는 하드 게이트(제출 불가), 50~199자는 소프트 품질선(제출 가능 + 정밀도 제한 배지)이다.

0자 · 브랜드 진단으로 제출 가능 (버튼 활성 + 모드 안내)

0자(공백 제외)

지금 제출하면 브랜드 진단(페르소나·USP·벤치마크)으로 생성됩니다. 상세페이지 카피를 넣으면 약기법 감사·문법 점수까지 포함됩니다.

1~49자 · 하드 게이트 (제출 잠금)

21자(공백 제외) 최소 50자 이상 콘텐츠가 필요합니다

콘텐츠는 50자 이상이어야 합니다. 비우고 제출하면 브랜드 진단으로 생성됩니다.

50~199자 · 소프트 품질선 (제출 가능)

62자(공백 제외) 정밀도 제한 · 200자 이상이면 더 정밀해집니다

“비우면 브랜드 진단”은 잠금 해제가 아니라 다른 상품이다. 그래서 0자 상태의 안내는 경고가 아니라 모드 예고다. 정밀도 제한은 잘못이 아니라 표본이 얇다는 고지라 경고색을 쓰고 버튼을 열어 둔다. 이 배지는 리포트 상단(REPORT-02)까지 따라간다.

INPUT-03

URL 모드

3e · 형식 검증 · 실패 시 두 출구

URL을 입력했는데 http(s) 형식이 아니면 버튼이 잠긴다. 비어 있으면 잠기지 않는다(브랜드 진단). 추출 실패는 브랜드 진단으로 조용히 강등하지 않고 실패 처리 + 두 출구(텍스트 붙여넣기 / 콘텐츠 비우고 브랜드 진단)를 안내한다.

진단 대상 콘텐츠 (상세페이지 카피)

이미지 위주 상세는 텍스트가 적게 추출될 수 있습니다. 실패하면 텍스트 붙여넣기로 안내됩니다.

추출 실패 메시지에 두 출구가 들어 있다: “상세페이지 텍스트를 직접 붙여넣어 주세요 … 또는 콘텐츠를 비우고 제출하면 제품 정보 없이 브랜드 진단만 받으실 수 있습니다.” 사용자가 제품 진단을 명시적으로 요청했으므로 조용한 강등이 실패보다 나쁘다(PROCESS-02).
