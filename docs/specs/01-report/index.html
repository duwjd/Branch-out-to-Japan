① 진단 리포트 · 인터랙티브 프로토타입

일본 시장 진입 진단 리포트 — 화면 흐름

Claude Design v2 디자인을 동작하는 프로토타입으로 이식했습니다. 입력 → 처리 → 리포트(상단 고정 요약 + 3탭) → 보고용 슬라이드까지 실제로 클릭됩니다. 각 노드를 눌러 화면으로 이동하거나, 상단 입력 시작에서 처음부터 체험하세요.

예시 데이터 고지. 화면 속 종합점수(18/100)·약기법 판정·페르소나·일본어 카피는 정본 샘플(../01-report-sample-cica-ampoule.md · 가상 브랜드 HARUON CICA 앰플)의 UI 시연용 예시입니다. 실제 진단 결과가 아니며, 1장 시장 이미지는 타사 저작물의 분석 목적 소량 인용(출처 병기)입니다.

전체 플로우

파이프라인이 성공하면 곧바로 발행되고, 고객은 리포트를 열람한 뒤 보고용 슬라이드를 내려받습니다. 사람이 개입하는 단계는 없습니다(2026-07-16 검수 제거).

1 · 입력

진단 입력

브랜드(필수)·제품(선택)·이미지 업로드(권장)/텍스트 게이팅. “예시 채우기”로 바로 체험.

 → 2 · 처리

진단 처리

8단계 파이프라인 애니메이션(이미지 추출 포함). 파이프라인 성공 = 발행.

 → 3 · 리포트

리포트 열람

상단 고정 요약(표지+진단 요약) · 3탭 · 슬라이드 내보내기.

리포트 구조 — 상단 고정 + 3탭 하이브리드

표지(블록 0)와 진단 요약 헤더(블록 1: 종합점수·신뢰 배지·총평·최우선 재설계 Top 3)는 탭과 무관하게 항상 상단에 고정되고, 본문은 시장 → 진단 → 처방 3탭으로 나뉩니다(2026-07-22, 구 5탭 단일 스크롤 대체 — 결론이 항상 첫 화면에 있어 결재자 동선이 짧아짐). 블록 번호(blocksJson ID)·채점 산식·LLM 콜 계약은 불변이며 표시 구조·라벨만 재편했습니다(REPORT-14). 화면 라벨은 전부 평문이며 A~E·K1~K11 내부 코드는 노출하지 않습니다.

published ○

처리 실패

콜② 감사 실패 = 잡 실패. 발행 안 함.
