## 0. 30초 요약

- **대상:** 이미 일본에 진출했으나 매출 저조 · 인하우스(디자이너·마케터) 없이 외주로 하던 소규모 뷰티 브랜드.
  → 기존 문서의 **S2 세그먼트**이며, 과거 가설 **H5("그래서 리뷰가 쌓이냐" 불신)로 회수 실패**했던 구간.
- **핵심 돌파:** 이들의 진짜 병목은 상세 카피가 아니라 **리뷰(口コミ)가 안 쌓이는 것**. → **리포트가 리뷰 병목을 진단·설계하고, 구독(20만)에 리뷰 축적 운영을 편입**하니 정체됐던 S2가 지불의향으로 전환됨.
- **검증 결과(페르소나 시뮬 4회):**

| 상품 | 가격 | 검증 결과 |
|---|---|---|
| **Report** | 30만/1회 | P09·P11 **둘 다 "낸다"** ✅ |
| **Growth 구독** | 월 20만 | P11 **"간다(사인)"** · P09 **"리포트 후 거의 간다"**(정상 퍼널) ✅ |
| 컨택 대행 | 유료 add-on | 리소스 부재 브랜드용 옵션 |
| **Scale** | 월 80만 | 복수채널·상시·매칭·전담(상위 위임형) |

- **원칙:** 번역이 아닌 **재설계** · 증거 원칙(가짜 스탯·성과보장·가짜 리뷰·AI 이미지 오인 금지) 전 구간 준수.

## 확정된 상품 구조 (요금표)

- **Report 30만/1회** — 진단 9블록 + **리뷰 축적 병목 진단·설계** + 일본향 썸네일 1 + 상세 1건(각 요소 재설계 **근거 주석**).
- **Growth 월 20만** — 상세 재설계 분기 1건 · 썸네일/피드 월 15~30건 · **리뷰 축적 운영 "바로 실행 가능한 키트"**(기프팅 브리핑+스테마 표기 가이드 · 리뷰 유도 문안 · 나노 인플루언서 매칭 리스트+컨택 템플릿 · 운영 SOP) · **첫 4주 성과 확인 리포트**(리뷰·CTR·전환 전후) · 월 1회 라이트 체크 + 분기 심층 재점검.
- **컨택 대행 add-on(유료)** — 실제 인플루언서 컨택·발송 대행. 실비 별도(1인당 10만원대·초기 월 3~5명 권장).
- **Scale 월 80만** — 복수채널·대량 + 약기법 상시 모니터링 + 일본 기업 매칭 + 전담 검수자 + 고급 성과 분석.

## 단계별 프롬프트 플레이북

> 사용법: 각 STEP의 "프롬프트"를 복사해 AI에 입력. `[]`는 상황에 맞게 교체. 앞 단계 산출물을 첨부/링크하면 정확도가 올라갑니다.

### STEP 1 · 세그먼트 분리 + 입점 후 리포트 스펙
- **목적:** 대상을 "입점 전(S1)"과 "입점 후·매출 저조(S2)"로 나누고, S2용 리포트 스펙을 만든다.
- **프롬프트:**
```
우리 서비스 대상을 두 세그먼트로 나눈다: ① 일본 진출 고려(입점 전), ② 이미 진출했으나 매출 저조(입점 후).
기존 입점 전 리포트 스펙(첨부)을 베이스로, 입점 후 세그먼트용 변형 스펙을 델타(변경점) 중심으로 작성해줘.
입점 후는 리뷰가 0일 수도 있는 점, 통증이 "왜 안 팔리는지"인 점을 반영하고, 증거 원칙(가짜 스탯·성과보장 금지)과 3축 구조(진단/스튜디오/운영)를 지켜줘.
```
- **결과·결정:** S2 = 기존 H5 실패 세그먼트임을 명시. 프레이밍을 "판매 중 상세 재진단"으로. → `docs/specs/02-report-spec-postentry.md`

### STEP 2 · 30만 산출물 정의 (리포트 + 썸네일 + 상세)
- **목적:** "진단만 하고 해결은 또 돈?" 마찰을 없애기 위해 실물 산출물을 30만에 포함.
- **프롬프트:**
```
30만 리포트에 "실물 산출물"을 포함하려 한다. 리포트 + 일본향 썸네일 1개 + 상세페이지 1건.
크몽의 저가 일본어 상세 제작(1~2만원대)과 구별되게 하려면 무엇이 포함돼야 하는지, 그리고 마진·포지션(종합대행 금지)이 무너지지 않는 경계를 제안해줘.
```
- **결과·결정:** 산출물은 "근거가 붙은 실물"(재설계 근거 주석). 저가 제작과의 차별을 legible하게. 30만은 구독 전환용 트라이얼 위상.

### STEP 3 · 페르소나 시뮬로 지불의향 1차 검증
- **목적:** 실제 타깃이 30만을 낼지, 어디서 망설이는지 수집.
- **프롬프트:**
```
[정체형 페르소나 P09/P11/P12 + 대조군 P19]에게 아래 30만 패키지 제안(고객이 보는 화면)을 제시하고,
① 첫인상·신뢰도 ② 30만 지불의향(낸다/조건부/안냄)과 망설이는 지점 ③ 구독 전환 의향 ④ 정직 고지 반응 ⑤ 크몽 대비 납득 ⑥ 세그먼트 적합 ⑦ 하나만 바꾼다면 을 캐릭터로 답하게 해줘.
[제안 화면 텍스트 첨부]
```
- **결과·결정:** 전원 "조건부". **최대 발견 — 진짜 병목은 리뷰 축적(H5 재현).** → `docs/research/시뮬-postentry-재시뮬-2026-07-08.md`

### STEP 4 · 리뷰 병목(H5) 대응 블록 추가
- **목적:** 리포트가 리뷰 병목을 정면으로 다루게 한다(진단·설계, 실행·보장은 안 함).
- **프롬프트:**
```
페르소나가 "진짜 문제는 리뷰가 안 쌓이는 것"이라 지적했다. 리포트에 "리뷰 축적 병목 진단·설계" 블록을 추가해줘:
① 왜 리뷰가 안 쌓이나 진단(유입/유도장치/이탈/신뢰공백) ② 초기 리뷰 확보 경로 설계(샘플링→나노→UGC→랭킹, 기프팅 브리핑+스테마 가이드).
단 리뷰 수·평점은 보장하지 않는다(스테마 규제·성과보장 금지). 실행은 구독으로 연결.
```
- **결과·결정:** 신규 블록으로 리포트에 편입. 리뷰 병목이 곧 구독 전환 근거가 됨.

### STEP 5 · 산출물 프로토 제작 (반자동 조립 + 근거 주석)
- **목적:** "돈값" 품질이 나오는지 + 크몽 차별화가 실물로 드러나는지 검증(Q4).
- **프롬프트:**
```
[샘플 제품: CICA 앰플]로 30만 산출물 프로토를 만들어줘. 진단 파이프라인 산출물(재작성·USP·페르소나·합법 대체표현)을 "반자동 조립"하고,
각 요소에 재설계 근거 주석(약기법 K코드 회피·코퍼스 근거·구매반대이유·USP)을 붙여줘. 일본어 옆에 KR 역문 병기.
단일 HTML·외부의존 0·증거원칙(확인된 수치만, 미확인은 플레이스홀더).
```
- **결과·결정:** 반자동화 ~60~70%, 근거 주석으로 크몽 차별화 legible. → `design/wireframes/deliverable-proto-cica.html`

### STEP 6 · 이미지 (AI 포토리얼 + 일본 플랫폼 법규 경계)
- **목적:** 실물 완성도 확보 + "AI 이미지를 실제 상세에 써도 되나?" 리스크 확인.
- **프롬프트(조사):**
```
AI 생성 이미지를 Qoo10·@cosme 실판매 상세페이지에 써도 되는지, 일본 플랫폼 규정 + 薬機法·景表法·스테마 규제 관점으로 조사하고 우리 산출물에 적용되는 경계를 정리해줘.
```
- **결과·결정:** AI 이미지 금지는 아니나 **실사와 동일 기준.** **제품 본체 컷=실물 필수**, 배경·연출만 AI, 효능·전후·가짜리뷰는 금지. → 우리 산출물 이미지는 "구성·카피 재설계 제안(데모)", 실판매 제품컷은 브랜드 실촬영. → `docs/research/AI생성이미지-일본플랫폼-법규-2026-07-08.md`

### STEP 7 · 구독 플랜 재구성 (20만에 리뷰 운영 편입)
- **목적:** 재시뮬에서 "20만 플랜에 리뷰가 없다"는 병목 해소.
- **프롬프트:**
```
현 랜딩 요금표(첨부)는 리뷰 축적이 상위 티어(80만)에만 있어 입점 후 세그먼트의 구독 전환이 막힌다.
리뷰 축적을 "설계=Report(30만)/실행=Growth(20만)"로 배치하는 플랜 재구성안을 델타로 제안해줘.
확정값: 구독 월 20만, 실비 별도, 실행 경계(키트 vs 대행)와 조기 성과 확인점 포함.
```
- **결과·결정:** Growth 20만에 리뷰 운영 키트 + 첫 4주 성과 확인 편입. 컨택 대행은 유료 add-on. → `docs/specs/03-구독플랜-재구성-제안.md`

### STEP 8 · 랜딩 요금표 반영 + 가상 스탯 제거
- **목적:** 확정 플랜을 랜딩에 반영하고, 페르소나가 불신한 가짜 성과 수치를 제거.
- **프롬프트:**
```
확정 플랜(첨부)을 랜딩 요금표에 반영하고, 가상 성과 스탯(체류+41%·CVR+38%·β 12개 브랜드 등)을 전부 제거해줘.
숫자는 근거 있는 실측(코퍼스 304건·효능 56항목)만 유지, 없으면 "실측 준비 중" 정직 표기. 새 가짜 숫자 생성 금지.
```
- **결과·결정:** 요금표 재구성 + 가상 스탯 삭제 + 히어로 성과약속 중립화. → `persona-simulation/landing/index.html`

### STEP 9 · 재시뮬로 전환 확인 (2~4차)
- **목적:** 각 개선이 실제 지불의향을 움직이는지 확인.
- **프롬프트:**
```
[P09·P11]에게 지난번 그들이 지적한 지점을 반영한 업데이트 버전을 제시하고, 지불의향이 "조건부→낸다/간다"로 이동했는지, 남은 병목은 무엇인지 캐릭터로 답하게 해줘.
```
- **결과·결정:** 30만 both "낸다", 20만 P11 "간다"·P09 "리포트 후 거의 간다". **S2 전환 루프 검증 완료.**

## 기획자 검토 포인트 (조정 가능한 값)

이번에 **합리적 기본값**으로 채운 것들 — 실제 원가·운영 감안해 조장이 확정 요망:
- 콘텐츠 **월 15~30건** 편차 기준(무엇으로 15 vs 30이 갈리나)
- 나노 인플루언서 **후보 리스트 월 10명 내외** 규모
- 실비 **1인당 10만원대 · 초기 월 3~5명 권장** — 실제 단가·표준 인원
- 점검 주기 **월 1회 라이트 + 분기 심층**
- 구독 20만의 **원가·마진**(리뷰 운영 키트 편입 시 인건비) — 반자동 조립으로 억제 전제
