# 05 · 팀 합의 필요 질문

> 팀MVP 진행 전 합의가 필요한 열린 질문. 결정되면 [[decisions/DECISIONS]] 에 기록한다.

## 1. 초기 타깃을 어디로 좁힐 것인가?
후보: 한국 코스메틱 브랜드 / 뷰티 디바이스 / 병원·피부과 / Qoo10 Japan 준비 브랜드 / 인스타 기반 소형 뷰티.
→ 추천: **한국 코스메틱·뷰티 브랜드** (니즈가 가장 명확). [[03-personas]]

## 2. 무엇을 직접 만들 것인가?
초기엔 툴을 만들지 않는다. 대신 진단 체크리스트, 샘플 리포트, 일본향 카피 개선안, 콘텐츠 기획안, 광고 소재 샘플, 랜딩페이지, 신청 폼을 만든다.

## 3. 어떤 툴을 내부 제작 도구로 쓸 것인가?
- ChatGPT / Claude: 진단·카피·리포트 초안
- Canva / Figma: 광고 이미지·리포트 디자인
- Carat / CapCut 등: 광고 영상 샘플
- Notion / Google Docs: 리포트 전달
- Tally / Google Form: 신청 폼
- Meta Ads: 인스타 광고 테스트
- YouTube Shorts / Instagram Reels: 콘텐츠 반응 테스트

## 4. 팀MVP 성공 기준은? → [[02-product-spec]] 지표 참고

## 5. 유료 상품 최소 성공 기준은?
예: 2주 내 무료 진단 신청 10건+, 상담 전환 3건+, 유료 결제(의사) 1건+, 반복 문제 문장 3개+.

## 6. (실행) Phase 1 리드 저장 방식
외부 폼서비스(Tally/Formspree) vs 자체 API route. → 기본안: 빠른 검증을 위해 외부 폼서비스. [[04-roadmap]]

## 7. (실행) 디자인 워크플로우
코드 우선(Tailwind)으로 시작. Figma MCP 연동은 추후 결정 시 `designer` 에이전트에 도구 추가.
