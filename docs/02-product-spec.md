## 커머셜 모델 (검증 가설 — 2026-07-06 시뮬레이션 반영 개정)

> 아래 퍼널은 **시뮬레이션 1·2차 기반 가설**이다 (근거·검증 계획: [[06-유저니즈-비즈니스-가설]] §4·5). 서비스 반영 방식은 **개발·UT(8월) 이후 확정** ([[decisions/DECISIONS]] 미결정 항목).

### 퍼널: Free → Report → Growth (→ Scale)

| 단계 | 상품 | 가격 가설 | 역할 · 근거 |
|---|---|---|---|
| **Free** | 약기법 간이 체커(승격안) + 체크리스트 — URL/캡션 입력 → 위반 표현 검출 + 조항 근거 | 무료 | 즉시 가치 증명 + Report 영업. "결과물 보고 판단"(17/20)에 대한 답. 근거가 법이라 AI 불신 우회 |
| **Report** | 진단 리포트 + **실물 산출물 2종(썸네일1 + 상세페이지1) 포함** — 문법 진단·NG/OK 재작성·페르소나·USP | **30만** | **전환 허브 = 구독 획득용 유료 트라이얼**(마진 상품 아님). 실물은 designer 반자동 조립(`deliverable-proto-cica.html`). S2는 퍼널 분해·리뷰 병목 진단 추가([[specs/02-report-spec-postentry]]) |
| **Growth** | 콘텐츠 제작 + **리뷰 축적 운영 편입** — 썸네일·피드 월 15~30건 + 상세 재설계 분기 1건 + 리뷰 운영 키트(기프팅·스테마 가이드·나노 인플루언서 매칭·SOP) + 첫 4주 성과 리포트 | 월 20만 + **첫 달 진단비 공제** | 리뷰 축적을 Scale→Growth로 하향(P11 "Growth에 리뷰 없다"=전환 병목). 실비 별도, 실제 컨택·발송=**컨택 대행 유료 add-on** → [[specs/03-구독플랜-재구성-제안]] |
| **Scale** | 복수 채널·대량 운영 전담 — 약기법 상시 모니터링 + 일본 기업 매칭 + 전담 검수자 + 고급 성과 분석 | 월 80만 | 상시·볼륨·매칭·전담으로 차별. 성과판별 β는 검토 |

> **증거 원칙:** 가상 스탯·익명 사례 금지(시뮬 2차 최대 이탈 트리거). 신뢰 장치는 ①샘플 리포트 실물 ②채점 기준 공개 ③벤치마크 분석 공개.
>
> **세그먼트 2분리(2026-07-08 확정):** S1(입점 전)=[[specs/01-report-spec]] / **S2(입점 후·매출저조)=[[specs/02-report-spec-postentry]]**. S2는 재시뮬로 유료 전환 검증(강등 2차타깃→승격). 구독 재구성·AI 이미지 경계는 [[decisions/DECISIONS]].

> (구) 월간 운영 산출물 목록 — 월간 콘텐츠 캘린더(일본향 주제 20개)·광고 소재 월 4~8개·숏폼 콘티 월 4개·일본어 카피·성과 리뷰·개선안 — 은 Growth 구독 스펙 설계 시 참고 자료로 유효.

## MVP 성공 기준 (검증 지표)

| 지표 | 판단 기준 |
|---|---|
| 광고 클릭률 | 문제 문구에 반응하는가 |
| 랜딩 신청률 | 제안이 이해되는가 |
| 무료 진단 신청 수 | 관심이 충분한가 |
| 상담 전환 수 | 문제의 긴급도가 있는가 |
| 유료 전환 | 실제 사업화 가능성 |

유료 상품 최소 성공 기준(예): 2주 내 무료 진단 신청 10건+, 상담 전환 3건+, 유료 결제(의사) 1건+, 반복되는 고객 문제 문장 3개+ 확보.

빌드 서비스는 위 지표에 더해 **UT 기반 검증(8/1~8/3)** — 실제 사용자가 3축 한 사이클(입력 → 리포트 → 콘텐츠 생성)을 완주하는지, 산출물이 "일본에서 통할 것 같다"는 신뢰를 주는지 — 로 확인한다.
