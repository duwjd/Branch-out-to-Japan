## 커머셜 모델 (검증 가설 — 2026-07-06 시뮬레이션 반영 개정)

> 아래 퍼널은 **시뮬레이션 1·2차 기반 가설**이다 (근거·검증 계획: [[06-유저니즈-비즈니스-가설]] §4·5). 서비스 반영 방식은 **개발·UT(8월) 이후 확정** ([[decisions/DECISIONS]] 미결정 항목).

### 퍼널: Free → Report → Growth (→ Scale)

| 단계 | 상품 | 가격 가설 | 역할 · 근거 |
|---|---|---|---|
| **Free** | 약기법 간이 체커(승격안) + 체크리스트 — URL/캡션 입력 → 위반 표현 검출 + 조항 근거 | 무료 | 즉시 가치 증명 + Report 영업. "결과물 보고 판단"(17/20)에 대한 답. 근거가 법이라 AI 불신 우회 |
| **Report** | 진단 리포트 — 일본 문법 진단 점수 + NG/OK 재작성(Before/After) + 페르소나·USP | **30만** | **전환 허브.** 시뮬 2차에서 17/20 수용, 가격 저항 사실상 소멸. 전원이 "Report 먼저 → 구독 판단" 경로 |
| **Growth** | 시즌 구독 — 시즌 캘린더 기반 기획 + 월 콘텐츠 패키지(채널 배분 명시) | 월 20만 + **Report 비용 첫 달 공제 브릿지** | 장벽은 금액이 아니라 "결과물 확인 전 월정액" 구조 → 브릿지로 해소. 갱신 근거 = 메가와리·코프레·UV 시즌 사이클 |
| **Scale** | 운영 확장(리뷰 메커니즘·기업 매칭) | 추후 | 시뮬 2차에서 전원 논외 — Report·Growth 검증 후 재설계 |

> **증거 원칙:** 가상 스탯·익명 사례 금지(시뮬 2차 최대 이탈 트리거). 신뢰 장치는 ①샘플 리포트 실물 ②채점 기준 공개 ③벤치마크 분석 공개.

<details>
<summary>이력 보존 — 구버전 3단계 모델 (2026-07-02, 시뮬레이션 이전)</summary>

- 1단계 무료 "일본 진출 콘텐츠 5분 진단" (입력: 브랜드/URL/인스타/목표/고민 → 위험도 5개·문구 예시 3개·방향 3개)
- 2단계 유료 스타터 "생존자들 Starter Kit" 49/79/120만 (메시지 진단·카피 개선·기획 10·광고 카피 5·콘티 3·소재 1~3·실행 우선순위)
- 3단계 월간 운영 "Japan Content Sprint" 월 150/200/300만

→ 시뮬 1차에서 가격 구간 불명확·견적 재현 우려, 2차에서 Report 30만 관문 확인으로 개정. 무료 입력 항목 표는 Free 체커 스펙의 기반으로 유효.
</details>

> (구) 월간 운영 산출물 목록 — 월간 콘텐츠 캘린더(일본향 주제 20개)·광고 소재 월 4~8개·숏폼 콘티 월 4개·일본어 카피·성과 리뷰·개선안 — 은 Growth 구독 스펙 설계 시 참고 자료로 유효.

## MVP 성공 기준 (검증 지표)

| 지표 | 판단 기준 |
|---|---|
| 광고 클릭률 | 문제 문구에 반응하는가 |
| 랜딩 신청률 | 제안이 이해되는가 |
| 무료 진단 신청 수 | 관심이 충분한가 |
| 상담 전환 수 | 문제의 긴급도가 있는가 |
| 유료 전환 | 실제 사업화 가능성 |

유료 상품 최소 성공 기준(예): 2주 내 무료 진단 신청 10건+, 상담 전환 3건+, 유료 결제(의사) 1건+, 반복되는 고객 문제 문장 3개+ 확보.

빌드 서비스는 위 지표에 더해 **UT 기반 검증(8/1~8/3)** — 실제 사용자가 3축 한 사이클(입력 → 리포트 → 콘텐츠 생성)을 완주하는지, 산출물이 "일본에서 통할 것 같다"는 신뢰를 주는지 — 로 확인한다.
