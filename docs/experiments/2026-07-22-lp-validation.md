# 실험 · 검증 랜딩(/lp) — MVP 실효성 조사

> 작성 2026-07-22. 목적: KGLOW MVP가 실제 수요(구매·필요성)를 갖는지, 유튜브 설명 영상 → 랜딩 → 상담/자료 신청 퍼널로 측정한다.
> **임계선은 결과가 나오기 전에 못 박는다.** 사후에 기준을 바꾸면 검증이 무효다.

## 1. 가설
한국 뷰티 중소기업 대표·마케터는 "일본 진출 콘텐츠가 일본 고객에게 통할지"에 대한 사전 진단·재설계 수요가 있으며, **정가 30만원을 인지한 상태에서도** 상담을 신청할 만큼 지불 의향이 있다.

## 2. 측정 설계
- **주 지표 = 상담 신청 완료율** (고관여 리드 = 지불의향 프록시).
- **보조 지표 = 자료받기(무료 샘플) 신청** (상단 관심 = 호기심 신호).
- **가짜문(fake door):** 진단 리포트 **정가 30만원**을 노출한 뒤 "파일럿 참여 브랜드에 무료(전체 리포트)". 가격을 보고도 신청 = 지불의향 프록시. (스튜디오 크레딧은 오퍼에서 제거 — §4-2)
- **유입:** 유튜브 공개 배포(단일 채널). 소스는 `?utm_source=` / `?src=` 파라미터로 태깅(1차 `src=r1` / 2차 `src=r2`).

### 퍼널 이벤트 (`/api/track`)
| 이벤트 | 의미 |
|---|---|
| `pageview` | 랜딩 방문 |
| `cta_click` | 상담/자료 CTA 클릭(어느 CTA인지 태그) |
| `video_play` | 임베드 영상 재생(선택 계측) |
| `lead_submit` | 상담/자료 신청 완료 |

리드 원본은 `/api/lead`로 저장(store: Supabase 또는 `.data/leads.json`). 소스 태그가 각 리드에 함께 저장된다.

## 3. 성공 임계선 (사전 확정)
표본 하한: **타깃 방문 100 이상**(그 미만이면 판단 보류 — 유튜브 유입 부트스트랩 필요).

| 판정 | 조건 (타깃 방문 100 기준) |
|---|---|
| **강한 유효** | 상담 신청 ≥ 8건 (8%) |
| **유효** | 상담 신청 5~7건 (5~7%) → MVP 방향 유지, 오퍼·카피 개선 후 확대 |
| **약함** | 상담 2~4건 (2~4%) → 메시지/타깃/오퍼 재설계 후 재실험 |
| **무효** | 상담 ≤ 1건 → 수요 가설 기각 또는 채널 전면 재검토 |

- 자료받기(보조)는 신청이 많아도 **단독으로 유효 판정 근거가 되지 않는다**(공짜 관심 ≠ 지불의향). 상담 대비 비율로 "관심은 있으나 지불 저항"을 읽는 용도.
- 정성 판정 병행: 상담 신청자의 채널·규모·고통점이 타깃 가설과 맞는지(요구 적합도). 신청 1건이라도 대화에서 강한 니즈 확인 시 별도 기록.

## 4. 유튜브 단일 채널 리스크 (필수 보완)
니치 B2B라 오가닉 발견이 약하다. **표본 100 확보가 최대 리스크.** 최소 2개는 실행한다:
1. 검색 타깃 제목/설명(고통점 키워드 — "일본 Qoo10 안 팔릴 때" 등).
2. 설명란·고정댓글·엔드카드 랜딩 링크 3중 배치(+ `utm_source=youtube`).
3. 초기 유입 부트스트랩(소액 유튜브 광고 또는 지인·커뮤니티 시드로 첫 100뷰).

## 4-1. 배포 계획 — 무료 전체 리포트 단일 오퍼 (개정 2026-07-23)
**MVP 기간엔 오퍼 A/B(무료 vs 유료)를 하지 않는다.** 유료 결제 검증이 MVP 종료 후로 연기되어(아래), MVP 기간 오퍼는 **무료 진단 리포트 전체 제공 단일 오퍼**다. 이걸 두 번에 걸쳐 배포·개선한다.
- **1차 배포 (2026-08-01) — 무료 진단 리포트 전체 제공.** 역할 ①크리에이티브·채널 학습 ②리드 리스트·수요 신호 확보.
- **2차 배포 (2026년 8월 중순) — 무료 진단 리포트 전체 제공(보완).** 1차 학습을 반영한 개선본(랜딩·대본·리포트 품질↑). 오퍼 종류는 동일, 실행 품질만 올린다. 1차(`src=r1`)↔2차(`src=r2`) 소스 분리로 개선 효과 비교.
- **유료 결제 오픈은 MVP 프로그램 종료 후로 연기 (변경 2026-07-23).** 가격·PG(토스)·환불·계약·세금 등 검토할 사항이 많고 복잡. MVP 기간엔 실결제를 열지 않는다. 유료용 오퍼·합격선·대본은 그 단계에서 확정한다(기존 유료 엔딩 초안은 대본에 보존).
- **지불의향은 상담에서 정성으로.** 정가 30만원 fake-door 노출은 유지 — 가격 보고도 신청 = 지불의향 프록시. 상담에서 "정가 30만원이면 결제하실 생각 있으셨어요?"를 직접 확인하고, 파일럿 팀은 **케이스 스터디**로 확보해 유료 오픈 시 활용.

## 4-2. 무료 오퍼 정련 — "너무 많이 주면 신호가 오염된다" (2026-07-23 논의)
무료 오퍼는 원래 지불의향을 못 잰다. 많이 줄수록 "공짜라 눌렀다"가 늘어 신호가 더 나빠진다. 그래서:
- **스튜디오 크레딧 제거 (확정 2026-07-23).** 크레딧은 측정 불가한 무료 단계에 비용만 늘린다. 크레딧은 **유료 오픈(MVP 종료 후)의 지렛대**로 아낀다.
- **무료 ≠ 검증.** 무료 리드 수는 상단 관심 신호일 뿐. 지불의향은 **신청 → 짧은 상담 → 리포트 전달** 흐름의 상담에서 정성으로 캔다. 파일럿 팀은 **케이스 스터디**로 재활용해 유료 오픈 시 쓴다.
- **핵심:** 주는 양을 줄이는 게 아니라, **리드 한 명에게서 뽑는 정보량을 늘리는 것**이 목표.

### 결정 로그 (노션 팀 문서와 동기화)
> 팀 공유: 노션 "검증 랜딩(/lp) 실험 — 기획·결정" (MVP 허브)
- [x] **결정 A (확정 2026-07-23)** — 무료 오퍼 = **전체 리포트**(샘플 분기 안 함) · 스튜디오 크레딧 제거.
- [x] **결정 C (확정 2026-07-23)** — 훅·제목 확정. **1차 훅=질문 프레임 / 2차 훅=Before-After 실증 티저**(둘 다 '사는 이유' 프레임). 1차 제목 "일본 Qoo10, 번역은 끝냈는데 왜 안 팔릴까 — 상세페이지 '일본 고객 관점' 진단". 상세 `2026-07-23-youtube-script.md` §B·C.
- (이월) **유료 합격선** — 유료 오픈(MVP 종료 후) 단계에서 확정. 예시값 방문 100당 실결제 1~2건은 그때 재검토.

**메시지 원칙 (확정 2026-07-23):** **약기법은 '편리·안심' 보조 소구로만.** KR·JP 규제가 비슷해 신선하지 않고, "약기법 해결→잘 팔린다" 인과는 한국 셀러에게 설득력이 약하다. 주 소구는 **일본 고객 관점의 '사는 이유' 재설계**(매출 직결). 약기법은 "귀찮고 위험한 표현 규제를 우리가 안전하게 정리"라는 편의로만 등장시키고, 훅·제목·헤드라인에 올리지 않는다.
