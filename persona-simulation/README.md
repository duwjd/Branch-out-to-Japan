# persona-simulation — 페르소나 검증용 랜딩페이지

> **⚠ 이 폴더의 산출물은 실제 배포물이 아니다.**
> AI 합성 페르소나에게 제시할 자극물(stimulus)로, `docs/research/archive/페르소나-검증형-MVP-실행안.md`의
> "0차 검증"(광고 집행 전 사전 검증)을 위해 만들어졌다. 백엔드 없음, 폼은 가짜 제출.

## 폴더 구조

```
persona-simulation/
├── README.md                        ← 이 문서
├── landing/index.html               ← 랜딩페이지 (단일 자기완결 HTML, 외부 의존 0)
├── copy/persona-test-variants.md    ← 전체 카피 팩 + 변형 분해 이유 + 금지 포지션 검수 결과
├── screenshots/                     ← 변형 5종 × 데스크톱(1280px)/모바일(390px) 캡처 10장
└── results/                         ← 페르소나 반응 기록·분석표 보관용 (추후)
```

## 랜딩페이지 사용법

브라우저에서 `landing/index.html`을 바로 연다 (`file://` 지원, 서버 불필요).

| 파라미터 | 효과 |
|---|---|
| `?variant=A` ~ `?variant=E` | 히어로 변형 전환 (무효값·미지정 시 A) |
| `?clean=1` | 상단 운영자 툴바 숨김 — **페르소나에게 제시할 때 필수** |

- 변형은 **히어로 블록(아이브로우·헤드라인·서브카피·CTA 문구)만** 바꾼다. 나머지 섹션은 전부 공통.
  MVP 실행안 §7이 "제안 문구 5안 비교"이므로, 변수를 히어로에 고립시켜 문구 반응만 측정한다.
- 상단 툴바(A~E 버튼)로 새로고침 없이 전환 가능. 페르소나 제공용 캡처에는 반드시 `clean=1`.
- 폼은 제출 시 네트워크 요청 없이 완료 메시지로 교체된다 (다음 행동 의향 측정용 연출).

### 변형 5종 (MVP 실행안 §7)

| 변형 | 톤 | 헤드라인 요지 |
|---|---|---|
| A | 문제 자극형 | 어디서부터 바꿔야 할지 막막하신가요? |
| B | 결과 제안형 | 일본 고객이 이해할 수 있는 말과 콘텐츠로 바꿔드립니다 |
| C | 자동화 강조형 | 일본어 번역만으로는 부족합니다 |
| D | 리스크 감소형 | 광고비를 쓰기 전에 먼저 점검하세요 |
| E | 실무 대행형 | 실행 산출물을 한 번에 정리해드립니다 |

## 검증 지표 ↔ 페이지 섹션 매핑 (MVP 실행안 §10)

| 지표 | 측정 섹션 |
|---|---|
| 문구 이해도 | 히어로(변형), 차별점 선언 |
| 문제 공감도·긴급도 | "혹시 이런 고민 중이신가요?" 인용 4개 |
| 산출물 선호도 | Starter Kit 9종 목록 |
| 가격 수용도 | 가격 5단 (무료/30/50/100/월 100~250만 원) |
| 신뢰 장벽 | Before/After 재설계 예시, AI+전문가 검수 투명화, FAQ |
| 다음 행동 의향 | 무료 진단 신청 폼 (티어별 "이 구성으로 문의" 포함) |

의도적으로 **허위 사회적 증거(고객사 로고·실적·후기)를 넣지 않았다.** "이 정도 신뢰 요소일 때
페르소나가 무엇을 더 요구하는가" 자체가 측정 대상이다.

## 페르소나 시뮬레이션 절차

1. **텍스트 스냅샷 (주 방식)** — `?clean=1` 기준 페이지 전체 텍스트를 페르소나 프롬프트에 삽입.
   변형 간 차이는 히어로뿐이므로 "공통 본문 1벌 + 히어로 5벌" 구조로 토큰을 절약할 수 있다.
   히어로 5벌 원문은 `copy/persona-test-variants.md` §1에 정리되어 있다.
2. **스크린샷 (보조)** — `screenshots/variant-X-{desktop,mobile}.png`를 멀티모달 페르소나에게 제시.
   시각 위계·가격표 인상·모바일 가독성까지 물을 때 사용.
3. **질문 세트** — MVP 실행안 §8(문제 인식 / 제안 반응 / 가격 검증 / 신뢰 장벽) 그대로 사용.
4. **순서 편향 완화** — 페르소나 10~20개 × A~E 노출 순서를 랜덤화한다.
5. **기록** — 응답을 §10 지표 7종 기준 분석표로 `results/`에 정리한다.
   해석은 §10 해석 기준을 따른다 (예: 공감도↑·가격수용↓ → 저가 진단부터, AI 표현 거부감 → 표현 교체).

## 한계 (MVP 실행안 §13 — 결과 문서에 반드시 병기)

합성 페르소나는 실제 고객이 아니다. 실제 구매 행동을 예측하지 못하며, 모델이 그럴듯한 답을
만들 수 있다. 이 검증은 광고 전 사전 검토로만 사용하고, 최종 판단은 실제 고객 인터뷰·신청 폼·
유료 제안으로 확인한다.

## 스크린샷 재촬영 (macOS, Chrome)

```bash
CHROME="/Applications/Google Chrome.app/Contents/MacOS/Google Chrome"
URL="file://$(pwd)/landing/index.html"
for V in A B C D E; do
  "$CHROME" --headless=new --disable-gpu --hide-scrollbars --virtual-time-budget=3000 \
    --window-size=1280,7200 --screenshot="screenshots/variant-$V-desktop.png" "$URL?variant=$V&clean=1"
  "$CHROME" --headless=new --disable-gpu --hide-scrollbars --virtual-time-budget=3000 \
    --window-size=390,5300 --screenshot="screenshots/variant-$V-mobile.png" "$URL?variant=$V&clean=1"
done
```
