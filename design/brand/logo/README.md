# KGLOW 로고 — 자산 정본 & 사용 규칙

> **확정 2026-07-16.** 서비스명은 **KGLOW**, 로고는 이 폴더의 SVG가 정본이다.
> 이전 워드마크(`Japan Growth Studio.` 텍스트 + 코랄 마침표)는 **폐기**한다.
> 브랜드 전략·보이스·비주얼 서사는 [../index.html](../index.html), 디자인 시스템은 [../../design-system.md](../../design-system.md).

## 1. 네이밍

- 고객 대면 이름은 **KGLOW** 하나로 통일한다. 전부 대문자, 붙여 쓴다.
- **쓰지 않는 표기:** `KGlow` · `K-Glow` · `K GLOW` · `kglow`(도메인·핸들·파일명 제외).
- `Japan Growth Studio`는 **이름이 아니라 카테고리 서술어**로만 남는다.
  예: "KGLOW — 일본향 메시지 설계가 내장된 Japan Growth Studio". 단독으로 서비스명처럼 쓰지 않는다.
- 일본어 표기는 로마자 `KGLOW` 그대로 둔다. 가타카나 음차(`ケイグロウ` 등)는 만들지 않는다.

## 2. 자산

| 파일 | 구성 | 언제 |
|---|---|---|
| [kglow-logo.svg](kglow-logo.svg) | 구체 + 네이비 워드마크 (가로형) | **기본값.** 헤더·푸터·문서 표지·발표 자료 |
| [kglow-logo-onDark.svg](kglow-logo-onDark.svg) | 구체 + 흰 워드마크 | 어두운 배경 |
| [kglow-wordmark.svg](kglow-wordmark.svg) | `KGL⬤W` — O 자리를 구체로 치환 | 로고를 크게 쓰는 표지·굿즈 등 연출 자리 |
| [kglow-mark.svg](kglow-mark.svg) | 구체 심볼 단독 | 파비콘·앱 아이콘·정사각 아바타 등 가로형이 안 들어가는 곳 |
| [kglow-logo-mono.svg](kglow-logo-mono.svg) | 네이비 1도 (띠는 배경이 비침) | 1도 인쇄·각인·팩스 |
| [kglow-logo-mono-onDark.svg](kglow-logo-mono-onDark.svg) | 흰색 1도 | 어두운 배경 1도 |

코드에서는 SVG를 직접 import하지 말고 **[components/brand/Logo.tsx](../../../components/brand/Logo.tsx)** 의 `KglowLogo` · `KglowMark`를 쓴다.

```tsx
import { KglowLogo, KglowMark } from '@/components/brand/Logo';

<KglowLogo height={28} />                  {/* 기본 가로형 */}
<KglowLogo height={28} tone="onDark" />    {/* 어두운 배경 */}
<KglowMark width={32} height={32} />       {/* 심볼 단독 */}
```

파비콘은 [app/icon.svg](../../../app/icon.svg) — Next.js App Router가 자동으로 잡는다.

## 3. 색 — **로고 전용 고정색**

> ⚠ 아래 값은 **로고에만** 쓴다. 디자인 시스템의 브랜드색은 여전히 스티비 코랄 `#ff6464`이고,
> CTA·칩·아이콘버튼은 그대로 `#ff6464`를 쓴다 (`design/design-system.md`). 로고 색을 UI 토큰으로 승격하지 않는다.

| 역할 | 값 |
|---|---|
| 워드마크 네이비 | `#22304F` |
| 구체 밝은 면 (그라디언트) | `#FF8570` → `#FF6A57` |
| 구체 어두운 면 (그라디언트) | `#E85742` → `#D8402B` |
| 스우시 | `#FFFFFF` |

단색이 필요하면 네이비 `#22304F` 또는 흰색만 쓴다. 코랄 1도 로고는 만들지 않는다.

## 4. 여백 · 최소 크기

- **보호 여백:** 구체 지름의 **0.25배** 이상을 로고 사방에 비운다. 이 안에는 텍스트·괘선·이미지 가장자리가 들어오지 않는다.
- **최소 크기:** 가로형 **높이 20px**(인쇄 7mm) / 심볼 단독 **16px**(인쇄 5mm).
  16px 미만에서는 스우시가 뭉개지므로 심볼 대신 네이비 `K` 이니셜을 쓴다.

## 5. 금지

- 구체와 워드마크의 **비율·간격 변경**, 재배치 (세로형이 필요하면 이 문서를 갱신하고 자산을 추가한다 — 임의 조립 금지)
- **장평 왜곡** — 높이만 지정하고 폭은 비율에 맡긴다
- **다른 서체로 워드마크 재조판** (자산의 아웃라인이 정본)
- 워드마크를 **코랄로** 칠하기 / 구체를 **네이비로** 칠하기 (모노 자산 제외)
- 구체에 **그림자·베벨·광택** 추가, 스우시 곡률 수정
- 대비가 부족한 배경(코랄 틴트·사진의 밝은 부분) 위에 네이비 워드마크 얹기 → `onDark` 또는 흰 배경 박스를 쓴다

## 6. 자산 재생성

워드마크는 **Montserrat 700**(SIL Open Font License 1.1) 글리프를 패스로 아웃라인한 것이다.
아웃라인이 자산에 박혀 있어 **폰트 설치·번들이 필요 없고**, OFL의 임베딩·파생 허용 범위 안에 있다.

곡선·간격을 바꿔야 하면 이 문서의 값(캡 하이트 100 기준: 구체 지름 `1.62`, 마크–워드마크 간격 `0.42`,
스우시 윗변 `M -10,114 C 45,138 125,108 210,26` / 아랫변 `M -10,128 C 45,152 125,120 210,38`)을 갱신하고,
`components/brand/Logo.tsx`와 이 폴더의 SVG를 **함께** 고친다. 둘이 어긋나면 SVG가 정본이다.
