## 1. 컴포넌트 — ⚠️ 부분 해소 (버튼 완료 · 2026-07-09)

> **진행 로그(2026-07-10 · 라인 스타일):** DS Button에 **코랄 `Outline` 변형 신설**(`163:4` — border `interactive/default #ff6464` · text `coral/strong #d93636`; 기존 Secondary는 회색 라인이라 브랜드 코랄 라인이 부재했음). **아웃라인 버튼 4개**(샘플 리포트 보기·전체 리포트 받아보기 데스크톱+모바일) → `Button/Outline` 인스턴스화. + **stale `#f04e4e` stroke 8곳 전수 정리**(이전 컬러 수정이 fill만 처리하고 stroke를 놓쳤음): 아웃라인 버튼 4개는 인스턴스로, 강조 테두리 4개(plan-growth·after)는 `coral/base` 바인딩. 잔여 stale stroke 0.
>
> **진행 로그(2026-07-09):** 채움형 CTA **17개 전부 DS `Button`(Primary/Default `40:5`) 인스턴스로 교체 완료** — 헤더·히어로·분석시작·체크·요금제×3·submit + 모바일 대응 9개. 레시피: 인스턴스 생성 → 부모 인덱스에 삽입 → 내부 TEXT 라벨 세팅 + `WIDTH_AND_HEIGHT`(단일행) → 부모가 오토레이아웃이면 원본 `FILL`/`HUG` 복사, 절대배치면 x/y·크기 복원. 배경은 컴포넌트가 `interactive/default(#ff6464)`에 바인딩돼 컬러도 자동 정합.
>
> **미전환(구조 불일치 — 강제 시 콘텐츠 손상):** 카드(요금제/체커목업/리포트/Before·After 등 다양 구조 vs DS `Card 42:16`=2텍스트 고정) · 인풋(문의내용=textarea, DS `Input 42:2`=단일행 고정) · 배지(코랄 "추천"·"확인 필요" vs DS `Badge`=semantic 톤) · 텍스트링크 CTA 5개(이미 `coral/strong` 변수 바인딩됨). → 권장: DS 컴포넌트를 **슬롯/프로퍼티로 일반화**한 뒤 인스턴스화하거나, 토큰 바인딩된 스타일 프레임으로 유지.

## 3. 컬러 — ✅ 대부분 해소 (2026-07-09)

> **진행 로그(2026-07-09):** 인스턴스 외부 raw fill을 semantic 변수에 일괄 바인딩 → **바인딩 25→555개(96%)**. 텍스트 324(`text/primary`·`text/body`·`text/muted`·`text/on-primary`·`feedback/success`·`coral/strong`) + 서페이스 189(`bg/page`·`bg/surface`·`bg/tint`·`feedback/success`). 값 동일 → **시각 변화 0**. **잔여 21개 = 대응 변수 없는 오프토큰**(아래) → 토큰 신설 시 바인딩 가능: `#e6f6f0`(success-tint) ×4 · `#c9cbcf` ×12 · `#9a9c9f` ×2 · `#3a3b3e` ×1(다크푸터 divider) · `#202124` 서페이스 ×2(다크푸터 bg, `bg/inverse` 토큰 필요).

- **(적용 전 상태) Solid fill 576개 중 변수 바인딩 25개**(2026-07-09 CTA/브랜드/텍스트CTA 처리분)뿐, **나머지 551개 raw hex**. 값 자체는 대부분 토큰과 일치하나 **미바인딩**이라 토큰을 바꿔도 전파 안 됨.
- **오프토큰 색 4종** (토큰 세트 밖):

| hex | 횟수 | 위치 | 판정 |
|---|---|---|---|
| `#e6f6f0` | 4 | Before/After 개선 카드 배경(프레임) | success 계열 tint — 토큰 미정의(있는 건 `success #00b97c`뿐). 유지하려면 `success-tint` 토큰 신설 |
| `#c9cbcf` | 12 | 네비·푸터 링크 | handoff §9에 다크푸터 링크로 문서화(AAA)했으나 **토큰 아님** |
| `#3a3b3e` | 1 | FOOTER `divider` | 다크 표면용 — `divider #ebebeb`와 별개, 토큰화 안 됨 |
| `#9a9c9f` | 2 | 푸터 저작권 | 문서화(AA)됐으나 토큰 아님 |

- 요지: 오프토큰 자체는 소수·대부분 의도된 다크푸터용. **진짜 문제는 96%가 미바인딩**이라는 점.
