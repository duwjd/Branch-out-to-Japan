## 0. 공통 규칙 (모든 화면 적용)

### 카피 언어
- **본문 = 한국어 B2B.** 1차 고객은 일본 소비자가 아니라 **대일 수출 준비 한국 회사**.
- **일본어는 Before/After·약기법 예시에서만.**
- **미확정 수치 삽입 금지**("5분" 등)·**가상 통계 금지**(시뮬 최대 이탈 트리거).

### 반응형
| 브레이크포인트 | 폭 | 비고 |
|---|---|---|
| Desktop | 프레임 1200 / 콘텐츠 max **1185** | 기본 |
| Tablet | 480–768 | 2→1 컬럼 전환 |
| Mobile | 390 (문서 기준 <480) | 단일 컬럼, 햄버거 내비 |

### 컴포넌트 (발명 금지 · 기존 인벤토리 사용)
`Button`(Primary/Secondary/Text/Disabled) · `Text Link` · `Input`(+States: Default/Helper/Error/Success) · `Textarea` · `Card` · `Badge`(Success/Warning/Error/Neutral) · `Chip` · `Avatar` · `Icon`(placeholder 링) · `Accordion Item`. Figma 파일 `2조 생존자들`, node id는 `design/handoff-landing-design-system.md` 참조.

### ⚠ CTA 코랄 정책 (load-bearing)
| 규칙 | 값 |
|---|---|
| 기본 CTA 배경 | `primary #FF6464` + **large/bold 흰 텍스트만** |
| hover/pressed/focus | `primary-strong #D93636` |
| 작은 인터랙티브 텍스트·링크·소형 배지 | `#D93636` (코랄 배경+작은 흰 텍스트는 ≈2.9:1 **AA 미달 → 금지**) |
| tint surface / CTA band | `primary-tint #FFF8F8` |
| validation/error | `error #F0483C` (CTA 색과 명확히 구분) |
| 금지 | 파란 primary(`#0066FF`), 마케팅 accent를 product UI에 사용 |

### 타이포 (Pretendard Variable · 8종)
Display 32 / H1 24 / H2 20 / Subtitle 16 / Body 14 / Body Small 13 / Caption 12 / Micro 10. Hero 헤드라인만 확대 적용 가능(스타일 자체는 Display 32).

### 스페이싱·라디우스
Spacing 4·8·12·16·20·24·32·64 / Radius 6·8·12·9999 / Elevation: Floating `0 4 12 /.08`, Modal `0 8 24 /.12`.

### 택소노미 정본 (canonical · 전 화면 통일 · ux-review C3)
- **카테고리(리포트 채점 엔진 키 = 정본, 대표 1개)**: `skincare · makeup · suncare · cleansing` — 라벨 일·한 병기(스킨케어/スキンケア …). 벤치마크·E군 채점의 종속 키라 엔진 키가 정본.
- **포지셔닝 태그(별도 필드·복수·선택)**: 더마 · 클린·비건 · 남성 등 — 카테고리와 직교하므로 분리.
- **제품 분류(4종·전 화면 동일)**: `화장품 · 의약외품 · 건강식품 · 잘 모르겠음`. 리포트 엔진 건강식품 지원 여부는 [플래그](§8) — 미지원 시 체커에서 "리포트는 화장품·의약외품 대상" 분기.
- 효과: 온보딩 프로필 → 리포트 Tier0/1 **프리필**(중복 입력 제거).

### 상태 규약 (DESIGN.md §14 매핑 · 접근성)
| 상태 | 규약(DESIGN.md §14) | 이 세트 적용 |
|---|---|---|
| Empty | 이유 한 문장 + 액션 하나, 일러스트 없음 | 체커 위반 0건 배너 |
| Loading | JobCard 치수 스켈레톤(첫 페인트) / 하단 인라인 스피너(갱신), 오버레이 금지 | 체커 결과·폼 submit·업로드 `[디자이너]` |
| Error(폼) | 1px error 보더 + 13px 헬퍼, 실행 가능 한 문장 | 브랜드명·비번·로그인 실패·약관 미동의 |
| Error(네트워크) | 중앙 heading+body+retry | URL fetch 실패 배너 / 각 호출 실패 `[디자이너]` |
| Success | 인라인 flash(토스트 아님) | 비번 사용 가능·저장 완료 |
| Disabled | opacity 0.4, geometry 유지 + 이유 안내 | 체커 제품분류 미선택 CTA |
| Focus | 2px outline ring #FF6464 + 2px offset | 버튼·chip·input·tab `:focus-visible` |
| reduced-motion | `prefers-reduced-motion: reduce` → 모든 duration instant | 와이어프레임에 미디어쿼리 포함 |
| 접근성 | label-input(for/id)·chip=button(role/aria)·file=네이티브 input·배너 아이콘 aria-hidden·tablist ARIA | 전 화면 반영 |

> 배지 대비: NG/OK·판정 배지는 흰 텍스트 대비 미달(흰 on `#00B97C`≈2.6:1) → **tint 배경 + 진한 텍스트**로 교체(ux-review A5).

## 2. 무료 약기법 체커 (공개)

- **목적**: 전환 허브 **in-link** · 즉시 가치(근거가 "법" → AI 불신 우회) · **비로그인 가능**
- **연결**: in ← 랜딩 · out → 샘플 리포트 / 회원가입

### 요소 인벤토리
| 요소 | 역할 | 컴포넌트 |
|---|---|---|
| 문구 입력 | 일본어 카피/URL 붙여넣기 | Textarea |
| 제품 분류 선택 | 화장품/의약외품/건강식품/잘 모르겠음(정본 4종) | Chip=button(role=radio·필수) |
| 진단 CTA | 판정 실행 → **같은 화면 결과(#ckResult)** 렌더 | Button Primary(block) |
| 무료 안내 | "비로그인 3회 무료 → 가입 시 월 3회" | Banner(info) |
| 결과 리스트 | 표현별 NG/OK + 조항 각주 | Card·Badge(tint+진한 텍스트) |
| 업셀 패널 | 가입 직행 + 샘플 보기 병치 | Card(tint)·Button Primary/Secondary |

### 상태 (DESIGN.md §14 · [반영]=와이어프레임 예시 있음)
- **Disabled** [반영]: 제품 분류 미선택 시 CTA 잠금 + 이유("제품 분류를 선택하면 진단할 수 있어요").
- **결과** [반영]: NG/OK 배지 = **tint 배경 + 진한 텍스트**(대비 개선), 취소선 원문 + 조항 각주.
- **Empty** [반영]: 위반 0건 → "위반 0건 · 좋은 시작이에요" + 전체 진단 유도.
- **Error(URL fetch 실패)** [반영]: "URL을 가져오지 못했어요 · 직접 붙여넣기" 배너 + 재시도.
- **게이트(무료 소진)** [반영/일부]: "3회 모두 사용 · 가입하고 계속" 배너 / 익명 카운트 저장방식은 [플래그](§8).
- **Loading**: 판정 처리 스켈레톤/인라인 스피너 `[디자이너]`(§14 Loading).

|---|---|---|---|
| 0 품의 표지 | 종합점수(0~100)+고정가 퍼널표 | Card·큰 수치 | 전체 노출 |
| 3 약기법 감사표 | 문장별 불가/조건부/가능+조항+대체 | Table·Badge(verdict) | 전체 노출(핵심) |
| 7 NG/OK 병기 | Before/After + 한국어 역해설 | 병기 카드 | **부분 잠금(veil)** → 전체 진단 유도 |

### 주의
- intro 배너에 **"실제 카피로 생성한 견본"** 명시(샘플 가상성 지적 반영).
- 감사표 하단 면책 + **실명 검수자 서명** 위치 확보(정책 확정 시 채움).
- 감사표 **모바일 반응형**: 가로 스크롤 vs 카드 분해 `[디자이너 결정]`.

## 4. 요금 (공개)

- **목적**: 고정가 퍼널 · "견적 왕복 없이"
- **연결**: in ← 랜딩/샘플/체커 · out → 회원가입

### Tier 명세
| Tier | 가격 | 대상 | CTA |
|---|---|---|---|
| Free | 0원 | 처음, 감 잡기 | Button Secondary |
| Report | 30만/1회 | 방향 확인 | Button Secondary("샘플 먼저") |
| **Growth** (featured) | 월 20만 | 콘텐츠 제작 | Button Primary |
| Scale | 추후 공개 | 운영·매칭 | Button Secondary |

### 주의
- Growth **"첫 달 진단비 공제"** 명시 = "결과물 확인 전 월정액" 거부(전원 보류) 해소 장치.
- **추천 배지**: 소형 흰 텍스트라 `primary-strong #D93636` 배경 사용(코랄+작은 흰텍스트 금지).
- 하단 노트: "고정가 · 견적 왕복 없이 · 언제든 해지 · 품의용 PDF 제공".
- **A/B 검토**: featured를 Growth vs Report 어디에 둘지(Report가 전환 허브) `[팀 결정]`.

## 5. 로그인 / 회원가입 (인증 · 신규)

- **목적**: 리포트 발행/전체 진단 시점의 가입 유도. **가입 = 온보딩 시작**(마찰 최소화)
- **연결**: in ← 체커/샘플/요금 CTA · out(가입) → 브랜드 프로필 / out(로그인) → 대시보드

### 폼 필드·상태 명세
| 화면 | name | label | type | 필수 | 상태 시연 |
|---|---|---|---|---|---|
| 회원가입 | email | 이메일 | email | ✓ | **Helper**("로그인 ID로 사용") |
| 회원가입 | password | 비밀번호 | password(new-password) | ✓ | **Success**("사용 가능") |
| 회원가입 | brandName | 브랜드명 | text(organization) | ✓ | **Error**("브랜드명을 입력해 주세요") |
| 회원가입 | agree | 약관·개인정보 동의 | checkbox | ✓ | — |
| 로그인 | email/password | (탭 전환) | — | ✓ | Default |

- 검증: 네이티브 `checkValidity()` 기반, 필드별 Helper/Error/Success 텍스트.
- **탭 전환**: 로그인 ↔ 회원가입 (authtabs).
- 성공: 제출 후 상태 전환(가입→온보딩 라우팅).

### 미확정
- **소셜 로그인(Google/카카오) 제공 여부** → 미제공 시 divider 이하 삭제. 리드 저장 방식(외부 폼 vs 자체 API)과 함께 결정. → [../../docs/05-open-questions.md](../../docs/05-open-questions.md) #6

## 6. 브랜드 프로필 등록 (온보딩)

- **목적**: IA **공통 엔티티 "브랜드 프로필" 생성**(3축 공유) · 진단 전 자산 수집
- **연결**: in ← 회원가입 · out → 진단 리포트 입력폼(① Tier 0)

### 4단계 스텝
`① 브랜드 기본 › ② 채널·판매페이지 › ③ 제품 문서 › ④ 진단 요청` (스텝 인디케이터: done/on/기본)

### 필드 명세 (②단계 예시)
| 필드 | 필수 | 컴포넌트 | 비고 |
|---|---|---|---|
| 브랜드명 | ✓ | Input(id/label) | |
| 카테고리 | ✓ | Chip=radio(대표 1개) | **정본**: 스킨케어/メイク·선케어·클렌징(일·한 병기) |
| 포지셔닝 태그 | 선택·복수 | Chip=checkbox | 더마/클린·비건/남성 (카테고리와 분리) |
| 제품 분류 | ✓ | Chip=radio | 화장품/의약외품/건강식품/잘 모르겠음(정본 4종) |
| 한국 채널 | 권장 | Input ×2(**개별 label**) | 홈페이지/판매 · SNS |
| 일본 채널 | 선택 | Input | Qoo10·라쿠텐·아마존JP |
| 상세페이지 문서 | 선택 | **네이티브 file input + label** | PNG·JPG·PDF |
| 제품 관리 정보 | 선택 | Textarea | 상품명·용량·가격(JPY)·바코드 |
| 오프라인 이력 | 선택 | Input | |

### 주의 / 상태
- **필수 최소화**: 별표만 강제, 나머지 `(선택)` → 이탈 방지. 선택 항목 나중 보완.
- "왜 묻나요" 안내 카드(번역≠재설계, 맥락=USP 정확도).
- **프리필** [반영]: 완료 시 리포트 Tier0에 카테고리·제품분류·채널 프리필("브랜드 프로필에서 이어받음").
- **복귀** [반영/일부]: "N단계까지 작성했어요 · 이어서" 배너 / 복귀 단계 UX·업로드 진행/성공/실패는 `[디자이너]`.
